FTP-katalogbegrensninger
Denne siden lar deg konfigurere hvilke kataloger virtuelle servereiere og brukere med FTP-tilgang som kan se når du logger på via FTP. Som standard kan FTP-klienter få tilgang til hele filsystemet, inkludert kataloger som /etc og /home . Imidlertid kan dette gi bort informasjon som du foretrekker at brukerne ikke kan se.

Hver rad i tabellen på denne siden definerer den virtuelle serveren eller serverne som skal matche, og katalogen de er begrenset til. I kolonnen Apply to server and sub-servers kan du enten velge alle virtuelle servere, eller et enkelt domene (som automatisk inkluderer alle dets under-servere). Hvis det sistnevnte alternativet er valgt og boksen Alt unntatt server er merket av, vil begrensningen påvirke andre servere enn den som er valgt.

Kolonnen Restrict to directory bestemmer hvor samsvarende virtuelle servere er begrenset til når du logger på via FTP. Det mest brukte alternativet er Brukernes hjemmekataloger , som vil begrense domeneeieren til den virtuelle serverens hjem, og eventuelle FTP-brukere til sine egne underkataloger, vanligvis under homes .

I en standard Virtualmin-installasjon har hjemmets katalog for hvert domene 750 tillatelser, noe som forhindrer brukere fra andre domener tilgang til det, uavhengig av FTP-katalogbegrensninger. Så å låse FTP-brukere på denne siden tilfører ikke så mye sikkerheten til systemet ditt.

I tillegg gjelder ikke begrensningene for denne siden PHP-skript, CGI-programmer eller virtuelle servereiere som logger på via SSH. For øyeblikket er det ingen måte å låse dem på, selv om tillatelser for hjemmekatalog fortsatt gjelder.
