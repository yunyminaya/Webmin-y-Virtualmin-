Nom d'hôte du serveur DNS maître
Par défaut, Virtualmin utilisera le nom d'hôte du système dans les enregistrements NS et SOA principaux pour les nouveaux domaines DNS qu'il crée. Cependant, si le nom d'hôte n'est pas un nom DNS valide ou ne convient pas à vos besoins, cette option peut être utilisée pour entrer un autre serveur de noms principal. Il doit résoudre l'adresse IP accessible en externe de votre système ou l'adresse d'un système de serveur de noms esclave pour les domaines créés à l'aide de ce modèle.
