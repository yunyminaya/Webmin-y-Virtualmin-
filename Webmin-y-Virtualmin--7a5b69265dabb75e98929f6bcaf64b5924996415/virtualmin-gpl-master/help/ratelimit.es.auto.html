Limitación de tarifa de correo
Esta página le permite controlar la velocidad a la que el servidor de correo de su sistema aceptará nuevos mensajes. De manera predeterminada, cualquier script en el sistema o el remitente remoto con un inicio de sesión SMTP puede enviar correos electrónicos sin límites de velocidad, lo que puede ser un problema si un spammer secuestra un inicio de sesión o script. Esta característica le permite establecer un límite razonable, como 1000 mensajes por hora en todo el sistema.

Además de un límite global, también puede establecer tarifas de dominio por remitente. Estos anularán cualquier límite global y se pueden usar para otorgar a un dominio de confianza particular un límite superior o inferior al que se aplica a otros.
