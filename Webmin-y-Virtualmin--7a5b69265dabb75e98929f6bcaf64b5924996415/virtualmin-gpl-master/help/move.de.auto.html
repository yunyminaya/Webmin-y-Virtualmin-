Virtuellen Server verschieben
Durch das Verschieben eines virtuellen Servers wird die Domänenverwaltung geändert, die die Kontrolle darüber hat. Die eigentlichen Funktionen des Servers sind nicht betroffen, können jedoch jetzt von einem anderen Administrator verwaltet werden und gelten für alle Einschränkungen für Domänen, Aliase und Datenbanken für diesen neuen Administrator.

Auf dieser Seite können drei verschiedene Arten von Vorgängen ausgeführt werden, je nachdem, welche Art von Server Sie bearbeiten und was Sie damit tun möchten. Sie sind :

Verschieben Sie einen Subserver unter einen anderen übergeordneten Server
Wenn Sie einen Subserver bearbeiten, können Sie im Menü eine andere übergeordnete Domäne auswählen, um deren Besitz zu ändern. Die Website des Servers, die DNS-Domäne, Postfächer usw. bleiben unberührt, sie werden jedoch jetzt vom Eigentümer des neu ausgewählten übergeordneten Elements gesteuert. Das Ausgangsverzeichnis wird unter das Ausgangsverzeichnis des neuen übergeordneten Elements verschoben, und der Besitz aller Dateien wird in den Benutzer und die Gruppe des neuen übergeordneten Elements geändert.

Verschieben Sie einen übergeordneten Server unter einen anderen übergeordneten Server
Wenn Sie einen übergeordneten Server bearbeiten, können Sie ihn auf dieser Seite in einen Subserver konvertieren. Dies sollte mit Vorsicht erfolgen, da einige Einstellungen, die nur für übergeordnete Server vorhanden sind (z. B. Kennwort, Kontingente und Grenzwerte), verloren gehen, da sie jetzt vom neuen übergeordneten Server geerbt werden. Außerdem werden alle Unix-Benutzer, Unix-Gruppen oder Webmin-Benutzer, die der Domäne zugeordnet sind, gelöscht.

Konvertieren Sie einen Subserver in einen übergeordneten Server
Wenn Sie einen Subserver bearbeiten, haben Sie die Möglichkeit, ihn in einen übergeordneten Server zu konvertieren, der einem neuen Serveradministrator gehört. Ein Benutzername und ein Passwort für den neuen Administrator müssen eingegeben werden. Nach Abschluss des Umzugs gehören alle Dateien im neuen Ausgangsverzeichnis des Servers dem neuen Benutzer. Festplattenkontingente, Bandbreitenbeschränkungen und Serverbesitzerbeschränkungen für den verschobenen Server werden vom alten übergeordneten Server kopiert, können jedoch anschließend separat bearbeitet werden.

Ändern des Ziels eines Alias
Wenn der ausgewählte Server ein Alias eines anderen Servers ist, kann dieses Formular verwendet werden, um ihn auf ein anderes Ziel zu verweisen. Sobald der Umzug abgeschlossen ist, gibt der Webzugriff auf die Domain Inhalte vom neuen Ziel zurück und E-Mails an die Domain werden an die Postfächer des neuen Ziels weitergeleitet.
