Benutzerdefinierte Shells
Auf dieser Seite können Sie die Unix-Shells anpassen, die Postfachbenutzern und Administratoren virtueller Server zur Verfügung stehen. Die Shell wird von Servern wie ProFTPd und SSHd verwendet, um die für jeden Benutzer zulässigen Anmeldetypen zu steuern, z. B. E-Mail, FTP und Remote-Shell-Zugriff.

Wenn die in Virtualmin integrierten Standardeinstellungen ausgewählt sind, werden die verfügbaren Shells automatisch von Virtualmin bestimmt, was für die meisten Systeme normalerweise zufriedenstellend ist. Wenn Sie sie ändern möchten, wählen Sie unten Benutzerdefinierte Shells aus und nehmen Sie Änderungen an der Tabelle vor.

Für jede Shell können die folgenden Felder bearbeitet werden:

Aktiviert?
Dieses Kontrollkästchen bestimmt, ob die Shell beim Erstellen oder Bearbeiten von Benutzern sichtbar ist.
Pfad zur Shell
Dies muss den vollständigen Pfad zum Shell-Befehl enthalten, z. B. /bin/sh oder /bin/false .
Beschreibung
Dies ist der Titel, der im Menü angezeigt wird, wenn die Shell ausgewählt wird.
Administrator?
Aktivieren Sie dieses Kontrollkästchen, wenn die Shell für Unix-Benutzer von Administratoren virtueller Server verfügbar gemacht werden soll.
Briefkasten?
Aktivieren Sie dieses Kontrollkästchen, wenn die Shell für Mail- / FTP-Benutzer verfügbar gemacht werden soll.
Standard?
Bei genau einer Shell für Administratoren und einer für Postfächer muss dieser Eintrag aktiviert sein. Dies zeigt an, dass dies die Standardeinstellung für Unix-Benutzer dieses Typs ist.
Zugriff
Dieses Menü zeigt Virtualmin an, welche Art von Zugriff von dieser Shell gewährt wird. Wenn Sie Ihre eigenen hinzufügen, sollten Sie auswählen, ob FTP und / oder SSH zulässig sind oder nicht.
