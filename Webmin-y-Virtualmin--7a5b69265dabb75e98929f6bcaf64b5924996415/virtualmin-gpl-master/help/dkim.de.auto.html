Identifizierte Mail von DomainKeys
DomainKeys Identified Mail (DKIM) ist ein Protokoll zum Signieren ausgehender E-Mail-Nachrichten mit einer Signatur, die beweist, dass sie tatsächlich von der Domain des Absenders stammt. Dies kann von Empfänger-Mailservern oder Spam-Filtern verwendet werden, um Fälschungen von Absendern zu erkennen. Dies wird normalerweise von Spammern durchgeführt, um zu vermeiden, dass ihre Domain auf die schwarze Liste gesetzt wird.

Wenn DKIM mit Virtualmin aktiviert ist, wird Ihr Mailserver so konfiguriert, dass allen ausgehenden Nachrichten eine DKIM-Signatur hinzugefügt wird, und jeder DNS-Domäne auf Ihrem System wird ein öffentlicher Schlüssel hinzugefügt. Andere Mailserver können diesen Schlüssel dann verwenden, um die Signatur für weitergeleitete Nachrichten zu überprüfen und so sicherzustellen, dass die Adresse des Absenders nicht gefälscht wurde.

Wenn Sie über virtuelle Server verfügen, für die DNS an anderer Stelle gehostet wird, E-Mails jedoch von diesem System gehostet werden, können Sie deren Domänennamen in das Feld Zusätzliche zu signierende Domänen eingeben. Die Einträge im Feld DNS-Einträge für zusätzliche Domänen können dann manuell zu diesen Zonen hinzugefügt werden.
