Consentire la modifica della configurazione di PHP?
Quando questa opzione è impostata su Sì (come per impostazione predefinita), i proprietari di server virtuali sono autorizzati a modificare la loro configurazione PHP. Tuttavia, è possibile impostarlo su No per impedirlo, per motivi di sicurezza o per impedire ad amministratori inesperti di apportare modifiche errate.

Questo viene implementato rendendo il file ~/etc/php.ini del dominio modificabile solo dall'utente root Unix. Tieni presente che questo non è un metodo infallibile per impedire modifiche alla configurazione di PHP - il proprietario del dominio potrebbe comunque specificare un diverso file di configurazione modificando gli script del wrapper PHP in cgi-bin , eseguendo la propria copia di PHP o eseguendo script PHP come programmi CGI.
