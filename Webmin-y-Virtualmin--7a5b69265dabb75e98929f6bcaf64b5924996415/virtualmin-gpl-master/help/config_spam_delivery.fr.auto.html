Livraison par défaut du spam
Si SpamAssassin est activé pour les comptes d'utilisateurs, les messages jugés comme spam par le processeur peuvent être remis (avec marquage facultatif, configuré dans le module SpamAssassin), supprimés ou remis dans un dossier spécial nommé Spam dans le répertoire de messagerie des utilisateurs. Bien que les faux positifs soient extrêmement rares avec SpamAssassin dans sa configuration par défaut, il peut être judicieux de conserver les messages pendant une certaine période et de permettre aux utilisateurs de vérifier le dossier de spam à l'aide de la messagerie Web Usermin ou via les dossiers IMAP (si IMAP est disponible pour vos utilisateurs) . La valeur par défaut est de placer le spam dans le dossier Spam Maildir.
