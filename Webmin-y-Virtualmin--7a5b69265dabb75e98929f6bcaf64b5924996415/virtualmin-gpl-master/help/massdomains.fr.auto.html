Mettre à jour les serveurs virtuels
Cette page vous permet de mettre à jour les paramètres et les limites dans plusieurs serveurs virtuels à la fois, comme sélectionné sur la page précédente. Par défaut, rien ne sera modifié sur les serveurs sélectionnés jusqu'à ce que vous choisissiez un attribut ou une limite à modifier à l'aide des champs de cette page. Les paramètres modifiables sont répartis dans les catégories suivantes:

Changements de quota et de bande passante
Ces champs peuvent être utilisés pour modifier les quotas de disque pour les serveurs sélectionnés ou pour modifier leurs limites de bande passante. La modification des quotas n'a d'effet que sur les serveurs de niveau supérieur, car les sous-serveurs héritent de leurs quotas du domaine parent.

Modifications des fonctionnalités activées
Dans cette section, vous pouvez sélectionner les fonctionnalités à activer ou désactiver pour tous les serveurs sélectionnés. Ce sont les mêmes fonctionnalités qui peuvent être activées pour des serveurs individuels sur la page Modifier le serveur virtuel .

Modifications des limites du propriétaire
Ces champs peuvent être utilisés pour modifier les différentes limites qui s'appliquent aux propriétaires de serveurs virtuels de niveau supérieur, telles que le nombre de boîtes aux lettres, d'alias et de domaines qu'ils peuvent créer. Les modifications apportées ici n'ont aucun effet sur les sous-serveurs, qui tombent sous les limites imposées à leurs parents.

Modifications des capacités du propriétaire
Les champs de cette section contrôlent les capacités de Virtualmin qui sont disponibles pour les administrateurs des serveurs virtuels sélectionnés. Encore une fois, ils ne concernent que les serveurs de niveau supérieur.

Modifications des options PHP
Lorsqu'elle est disponible, cette section vous permet de modifier à la fois la version PHP par défaut et la méthode d'exécution des scripts PHP sur tous les serveurs sélectionnés. Il n'est cependant utile que sur les serveurs qui ont des sites Web.

Changements de configuration
Cette section est destinée à apporter diverses modifications à la configuration des serveurs sélectionnés, telles que la méthode utilisée pour supprimer le spam ou le shell de connexion de l'administrateur de domaine.

Il est également possible de désactiver temporairement les serveurs sélectionnés à l'aide du bouton Désactiver les serveurs en bas de la page. Lorsque vous l'utilisez, vous pouvez également saisir un motif de désactivation (tel qu'un non-paiement ) dans la zone de texte adjacente.

Pour réactiver plusieurs serveurs qui ont été désactivés sur cette page ou ailleurs, utilisez le bouton Activer les serveurs .
