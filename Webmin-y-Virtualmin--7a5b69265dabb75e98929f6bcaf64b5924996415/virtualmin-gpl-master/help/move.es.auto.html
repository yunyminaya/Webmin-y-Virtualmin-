Mover servidor virtual
Mover un servidor virtual cambia la administración del dominio que tiene control sobre él. Las funciones reales del servidor no se ven afectadas, pero ahora puede ser administrado por un administrador diferente y cuenta para cualquier límite en dominios, alias y bases de datos para ese nuevo administrador.

Esta página se puede utilizar para realizar tres tipos diferentes de operaciones, según el tipo de servidor que esté editando y lo que quiera hacer con él. Son :

Mover un servidor secundario debajo de un padre diferente
Al editar un subservidor, puede seleccionar un dominio primario diferente del menú para cambiar su propiedad. El sitio web del servidor, el dominio DNS, los buzones, etc. no se verán afectados, pero ahora estará bajo el control del propietario del nuevo padre seleccionado. Su directorio de inicio se moverá debajo del inicio del nuevo padre, y todos los archivos tendrán su propiedad cambiada al usuario y grupo del nuevo padre.

Mover un servidor padre debajo de un padre diferente
Al editar un servidor principal, puede convertirlo en un servidor secundario utilizando esta página. Esto debe hacerse con cuidado, ya que algunas configuraciones que existen solo para servidores principales (como la contraseña, las cuotas y los límites) se perderán, ya que ahora se heredan del nuevo padre. Además, se eliminará cualquier usuario de Unix, grupo de Unix o usuario de Webmin asociado con el dominio.

Convierta un servidor secundario en un servidor primario
Al editar un subservidor, tiene la opción de convertirlo en un servidor primario propiedad de un nuevo administrador del servidor. Se debe ingresar un nombre de usuario y contraseña para el nuevo administrador. Una vez completado el movimiento, todos los archivos del nuevo directorio de inicio del servidor serán propiedad del nuevo usuario. Las cuotas de disco, los límites de ancho de banda y los límites del propietario del servidor para el servidor movido se copiarán del padre anterior, pero posteriormente se pueden editar por separado.

Cambiar el objetivo de un alias
Si el servidor seleccionado es un alias de otro servidor, este formulario puede usarse para apuntarlo a un objetivo diferente. Una vez que se completa el movimiento, el acceso web al dominio devolverá el contenido del nuevo objetivo, y el correo electrónico al dominio se reenviará a los buzones del nuevo objetivo.
