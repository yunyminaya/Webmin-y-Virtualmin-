Резервное место назначения
Эти поля управляют местом записи резервной копии Virtualmin. Вы можете выбрать один из трех режимов назначения:
Локальный файловый каталог
В этом режиме резервная копия просто сохраняется в файле в системе, работающей под управлением Virtualmin. Если используется формат резервной копии Single archive file , необходимо ввести здесь имя файла, например /backup/domainname.tgz . Если используется любой из форматов One file per server , необходимо ввести каталог, например /backup .
FTP-сервер
Этот режим можно использовать для загрузки файла или файлов резервной копии с использованием протокола FTP на другой сервер. Необходимо ввести имя сервера, логин, пароль и целевой файл или каталог.
SSH-сервер
Этот режим похож на режим FTP, но резервная копия передается по протоколу SSH.
Если формат резервной копии — один файл, то местом назначения, которое вы здесь указываете, должен быть файл, который будет перезаписан или создан. Если создается один файл на домен, то местом назначения должен быть каталог, который уже существует.

Если резервное копирование выполняется на SSH- или FTP-сервер через нестандартный порт, вы можете ввести его в поле сервера, отделив от имени хоста двоеточием, например, backupserver.yourdomain.com:2222 .

При резервном копировании на сервер SSH, который поддерживает только аутентификацию на основе сертификатов, поле пароля необязательно. Однако пользователь root в системе должен иметь свой открытый ключ SSH в файле authorized_keys конечного пользователя.
