Copia de seguridad de MySQL en una sola transacción?
Cuando esta opción se establece en Sí , las copias de seguridad de MySQL se realizarán en una sola transacción para garantizar la coherencia. Sin embargo, esto puede consumir más RAM cuando se realiza una copia de seguridad de una base de datos grande y cambiante. También puede fallar con el error No se pudo ejecutar 'FLUSH TABLES WITH READ LOCK' en algunas versiones de MySQL o Amazon RDS. En este caso, debe cambiar esto a No en su lugar.
