Einschränkungen des FTP-Verzeichnisses
Auf dieser Seite können Sie konfigurieren, welche Verzeichnisse Eigentümer und Benutzer virtueller Server mit FTP-Zugriff sehen können, wenn sie sich über FTP anmelden. Standardmäßig können FTP-Clients auf das gesamte Dateisystem zugreifen, einschließlich Verzeichnisse wie /etc und /home . Dies kann jedoch Informationen preisgeben, die Benutzer lieber nicht sehen können.

Jede Zeile in der Tabelle auf dieser Seite definiert den oder die zu passenden virtuellen Server und das Verzeichnis, auf das sie beschränkt sind. In der Spalte Auf Server und Subserver anwenden können Sie entweder alle virtuellen Server oder eine einzelne Domäne auswählen (die automatisch alle Subserver enthält). Wenn die letztere Option ausgewählt und das Kontrollkästchen Alle außer Server aktiviert ist, wirkt sich die Einschränkung auf andere als die ausgewählten Server aus.

Die Spalte Auf Verzeichnis beschränken bestimmt, wo sich übereinstimmende virtuelle Server bei der Anmeldung über FTP beschränken. Die am häufigsten verwendete Option ist Benutzer Home - Verzeichnisse, die den Domaininhaber auf den virtuellen Server zu Hause begrenzen und alle FTP - Benutzer ihre eigenen Unterverzeichnisse, in der Regel unter homes .

In einer Standardinstallation von Virtualmin verfügt das Basisverzeichnis jeder Domäne über 750 Berechtigungen, wodurch verhindert wird, dass Benutzer aus anderen Domänen unabhängig von den Einschränkungen des FTP-Verzeichnisses darauf zugreifen. Das Sperren von FTP-Benutzern auf dieser Seite trägt also nicht so viel zur Sicherheit Ihres Systems bei.

Darüber hinaus gelten die Einschränkungen auf dieser Seite nicht für PHP-Skripte, CGI-Programme oder Besitzer virtueller Server, die sich über SSH anmelden. Gegenwärtig gibt es keine Möglichkeit, sie zu sperren, obwohl die Berechtigungen für das Ausgangsverzeichnis weiterhin gelten.
