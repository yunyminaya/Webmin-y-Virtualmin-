Nome do host do servidor DNS mestre
Por padrão, o Virtualmin usará o nome do host do sistema nos registros NS e SOA principais para os novos domínios DNS criados. No entanto, se o nome do host não for um nome DNS válido ou não for adequado para seus propósitos, essa opção poderá ser usada para inserir um servidor de nomes primário diferente. Ele deve ser resolvido para o endereço IP acessível externamente do seu sistema ou o endereço de um sistema servidor de nomes escravo para domínios criados usando este modelo.
