Caractère générique pour les bases de données MySQL supplémentaires autorisées
Ce champ sert à entrer un modèle générique MySQL qui accorde l'accès à des bases de données supplémentaires - peut-être quelque chose comme ${DB}_% . Pour un domaine comme example.com , cela serait étendu à example_% , ce qui permettrait d'accéder à toutes les bases de données commençant par example_ .

Étant donné que Virtualmin crée des règles d'accès MySQL pour toutes les bases de données qu'il crée, ce champ peut généralement être laissé vide. Il ne doit être rempli que si vous créez des bases de données supplémentaires via une autre méthode hors du contrôle de Virtualmin.
