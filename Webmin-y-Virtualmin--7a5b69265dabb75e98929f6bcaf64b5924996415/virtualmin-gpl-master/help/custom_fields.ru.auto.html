Настраиваемые поля
Пользовательские поля позволяют администратору включать дополнительные данные об учетных записях домена. Поля будут отображаться при редактировании домена. Они могут быть использованы практически для любого типа данных. Например, может быть полезно иметь дополнительную контактную информацию, прикрепленную к учетной записи домена, или поле примечаний, используемое, чтобы объяснить, почему домен отключен или имеет какую-то дополнительную функцию или ограничение помимо других доменов, созданных с тем же шаблоном. Если вы разрабатываете инструменты для взаимодействия с Virtualmin, эти поля могут быть полезны для других целей.

Для каждого поля необходимо ввести следующее:

Имя поля
Краткий код этого поля, который используется для ссылки на поле в шаблонах и сценариях после создания. Каждое поле должно иметь уникальное имя поля.

метка
Текст, который будет отображаться пользователю при вводе значения для этого поля. Если метка содержит; все, что после него будет использоваться в качестве всплывающей подсказки, которая появляется при наведении курсора на поле на странице «Редактировать виртуальный сервер».

Тип
Это определяет, какие виды значений могут быть введены для этого поля. Большинство говорят сами за себя, за исключением опции меню . Когда это выбрано, вы должны ввести путь к файлу в системе Virtualmin в соседнем текстовом поле. Каждая строка в файле должна содержать код и отображаемый текст для записи в меню для этого настраиваемого поля, разделенные пробелами.

В списке серверов?
Если для этого параметра установлено значение «Да» , столбец для этого настраиваемого поля будет отображаться на странице «Список виртуальных серверов» .
