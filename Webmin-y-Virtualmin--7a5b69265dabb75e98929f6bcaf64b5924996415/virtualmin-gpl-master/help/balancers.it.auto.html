Percorsi proxy
Il proxy consente di mappare un percorso URL nel sito Web di un server virtuale su uno o più URL di destinazione. Viene in genere utilizzato per rendere disponibili le applicazioni Ruby on Rails servite da più istanze Mongrel in un percorso URL nel sito Web del server. Può anche essere utilizzato per rendere disponibili altri server Web come Tomcat tramite un percorso URL sul tuo sito Web.

Ad esempio, è possibile mappare il percorso /radiant agli URL http://localhost:3001 e http://localhost:3002 , che sono le porte per due istanze Mongrel che servono la stessa applicazione in esecuzione sul sistema Virtualmin.

Solo i sistemi su cui è installato il modulo Apache mod_proxy_balancer possono mappare un percorso a più URL. Altri che supportano solo il normale modulo proxy di Apache possono inoltrare un percorso a un solo URL.
