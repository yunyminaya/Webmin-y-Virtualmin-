Transferir servidor virtual
Esta página pode ser usada para enviar um servidor virtual (e quaisquer sub-servidores ou aliases) para outro sistema executando o Virtualmin. O processo de transferência usando a funcionalidade de backup e restauração do Virtualmin e incluirá todo o conteúdo do diretório inicial, bancos de dados, registros DNS e outras configurações.

O nome do host ou o endereço IP do sistema de destino deve ser digitado no campo Nome do host do sistema de destino , e o campo Senha raiz do sistema de destino deve ser preenchido, a menos que você tenha configurado a autenticação SSH sem senha.

Por padrão, o domínio será simplesmente copiado para o novo sistema. No entanto, o campo Ação após a transferência pode ser usado para movê-lo, excluindo o servidor do sistema de origem. Isso é necessário se a origem e o destino compartilharem um servidor DNS em cluster ou forem servidores de nomes secundários um para o outro.
