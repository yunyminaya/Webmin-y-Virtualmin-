Разрешить редактирование конфигурации PHP?
Если для этого параметра установлено значение «Да» (как по умолчанию), владельцы виртуальных серверов могут редактировать свою конфигурацию PHP. Однако вы можете установить его на Нет, чтобы предотвратить это, либо по соображениям безопасности, либо для того, чтобы опытные администраторы не могли вносить неправильные изменения.

Это достигается тем, что файл ~/etc/php.ini домена редактируется только пользователем root Unix. Имейте в виду, что это не надежный метод предотвращения изменений конфигурации PHP, хотя владелец домена все еще может указать другой файл конфигурации, отредактировав сценарии оболочки PHP в cgi-bin , запустив собственную копию PHP или запустив сценарии PHP. как программы CGI.
