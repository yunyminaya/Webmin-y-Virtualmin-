Installation et configuration de BIND activées?
Cela active ou désactive la configuration du serveur de noms local. Si vous avez déjà configuré la gestion de domaine ailleurs, ou si vous ne créerez pas de nombreux nouveaux domaines (si, par exemple, vous utilisez Virtualmin principalement pour la génération de comptes de messagerie dans un ou un petit ensemble de domaines), vous pouvez choisir de le désactiver. Notez que même si vous souhaitez héberger des DNS primaire et secondaire sur des machines distinctes, vous souhaiterez toujours que cette option soit activée, afin que la fonction Cluster DNS Slaves puisse être utilisée pour propager automatiquement les nouvelles zones vers vos serveurs maîtres.
