Režim vykonávania skriptu PHP
Režimy vykonávania skriptov PHP definujú, ako server spracuje kód PHP.

Zakázaný režim znamená, že skripty PHP nie sú vykonávané serverom, ale namiesto toho sa s nimi zaobchádza ako s obyčajnými textovými súbormi. Tento režim možno použiť, keď sa nevyžaduje spracovanie PHP alebo keď sa kód PHP podáva ako súbory na stiahnutie.

FPM je preferovaný režim, ktorý ponúka vysokú účinnosť a pokročilé funkcie, ako je adaptívne riadenie procesov. Funguje ako samostatný FastCGI server, ideálny pre stránky s vysokou návštevnosťou vďaka svojim robustným výkonom.

FCGI je užitočný pre servery, kde nie je k dispozícii FPM. Podporuje tiež konfigurácie pre virtuálnych hostiteľov, hoci vo všeobecnosti nezodpovedá FPM z hľadiska škálovateľnosti a funkcií správy.

Režim CGI spúšťa každý skript PHP ako samostatný proces prostredníctvom protokolu CGI. Tento režim je menej efektívny ako FPM alebo FCGI kvôli réžii vytvárania nového procesu pre každú požiadavku. Zriedka sa používa v moderných nastaveniach kvôli jeho obmedzeniam výkonu.

mod_php integruje PHP priamo do webového servera Apache, čo umožňuje spúšťanie PHP skriptov v rámci vlastného procesu servera. V dôsledku problémov so zabezpečením a izoláciou a prerušením podpory a aktualizácií je však teraz zastaraný.
