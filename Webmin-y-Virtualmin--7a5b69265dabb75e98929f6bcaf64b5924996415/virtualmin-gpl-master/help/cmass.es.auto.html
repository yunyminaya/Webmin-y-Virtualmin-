Servidores de creación de lotes
Esta página le permite crear múltiples servidores virtuales a la vez desde un archivo de texto. Este archivo debe incluir un servidor por línea, y cada línea debe tener el siguiente formato:

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

Los significados de los campos son:

dominio
El nombre de dominio que debe crear el servidor virtual, como foo.com .
descripción
Una descripción legible para humanos, como el sitio web de Joe .
contraseña
La contraseña para el inicio de sesión de administración del servidor. Esto solo es necesario para los servidores principales.
nombre de usuario
El nombre de usuario de administración para el servidor. Si se deja en blanco, se selecciona automáticamente un nombre de usuario.
dominio principal
El nombre de dominio del servidor principal para crear este servidor. Si esto se deja vacío, el nuevo servidor se crea como padre mismo.
dirección IP
Para los servidores virtuales que no necesitan su propia IP privada, este campo se puede dejar en blanco. De lo contrario, debe ingresar una dirección no utilizada válida o la palabra allocate para que Virtualmin seleccione una por usted (si está configurada).
dominio alias
El nombre de dominio del servidor virtual para el que esto debería convertirse en un alias. Si se deja fuera, el nuevo servidor será de nivel superior o un subservidor.

Las cuotas, el ancho de banda y otros límites para los servidores creados se establecen en función de la plantilla que seleccione en este formulario. Las funciones de los servidores creados están determinadas por las casillas de verificación de funciones, al igual que en la página normal de creación de un solo servidor.

Si su sistema usa cuentas de revendedor, puede seleccionar tener todos los servidores virtuales creados bajo la propiedad de un revendedor. Esta opción solo está disponible para el administrador maestro.
