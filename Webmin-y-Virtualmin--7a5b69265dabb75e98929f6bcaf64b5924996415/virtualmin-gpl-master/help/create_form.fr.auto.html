Créer un serveur virtuel
Ce formulaire vous permet de créer un nouveau serveur virtuel de niveau supérieur, qui aura une nouvelle connexion d'administration associée. Lors de la création du serveur, vous devez saisir au moins le nom de domaine, la description et le mot de passe d'administration. Si plusieurs modèles de serveur sont définis sur le système, vous pouvez en sélectionner un dans le menu - les modèles déterminent diverses valeurs par défaut pour le nouveau serveur, telles que la configuration du site Web, les enregistrements DNS, etc.

Dans la section Options avancées, vous pouvez sélectionner un nom d'utilisateur d'administration, un nom de groupe et un nom de base de données différents pour le serveur, plutôt que les valeurs par défaut qui sont extraites de la première partie du nom de domaine. Une adresse e-mail de contact pour le serveur peut également être saisie - idéalement, il doit s'agir de l'adresse existante de la personne qui gérera le serveur, dans un domaine différent.

Pour sélectionner les fonctionnalités qui sont activées pour le serveur, utilisez les cases à cocher dans la deuxième partie de la section Options avancées . Certaines fonctionnalités dépendent d'autres et ne peuvent pas être sélectionnées seules - par exemple, la création de rapports Webalizer sur les statistiques d'un site Web dépend du fait d'avoir un site Web.

Au bas de la section Options avancées, vous pouvez choisir de transférer tout le trafic Web vers ce nouveau site vers une autre URL et de transférer tous les e-mails. Vous pouvez également choisir que le nouveau serveur virtuel utilise ses propres adresses IP privées, ce qui est requis pour SSL et l'hébergement FTP anonyme.
