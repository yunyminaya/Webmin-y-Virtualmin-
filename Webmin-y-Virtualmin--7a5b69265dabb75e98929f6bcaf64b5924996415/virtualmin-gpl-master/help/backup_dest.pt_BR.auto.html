Destino de backup
Esses campos controlam onde o backup do Virtualmin é gravado. Você pode escolher entre um dos três modos de destino:
Diretório de arquivo local
Neste modo, o backup é simplesmente salvo em um arquivo no sistema que executa o Virtualmin. Se estiver usando o formato de backup Single archive file , você deve digitar um nome de arquivo aqui, como /backup/domainname.tgz . Se estiver usando qualquer um dos formatos One file per server , você precisa digitar um diretório, como /backup .
Servidor FTP
Este modo pode ser usado para ter o arquivo de backup ou arquivos carregados usando o protocolo FTP para outro servidor. Você deve digitar o nome do servidor, login, senha e arquivo ou diretório de destino.
Servidor SSH
Isso é semelhante ao modo FTP, mas transfere o backup usando o protocolo SSH.
Se o formato de backup for um único arquivo, então o destino que você especificar aqui deve ser um arquivo, que será sobrescrito ou criado. Se estiver criando um arquivo por domínio, então o destino deve ser um diretório que já exista.

Se estiver fazendo backup em um servidor SSH ou FTP em uma porta não padrão, você pode inseri-la no campo do servidor, separada do nome do host por dois pontos, como backupserver.yourdomain.com:2222 .

Se estiver fazendo backup para um servidor SSH que suporta apenas autenticação baseada em certificado, o campo de senha é opcional. No entanto, o usuário root no sistema deve ter sua chave pública SSH no arquivo authorized_keys do usuário de destino.
