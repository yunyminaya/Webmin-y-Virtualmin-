Configurar o trabalho do Webalizer Cron para cada servidor virtual?
Quando essa opção é definida como Sim (como é por padrão), o Virtualmin criará um trabalho Cron separado para cada servidor virtual com o Webalizer ativado, a fim de gerar um relatório diariamente a partir de seus logs.

No entanto, em um servidor com um grande número de domínios, você pode alterar essa opção para Não e agendar manualmente o comando /usr/libexec/webmin/virtual-server/run-all-webalizer.pl usando o Cron. Isso garantirá que o Webalizer seja executado para cada domínio, em vez de potencialmente ter vários domínios processados ao mesmo tempo, além de fornecer mais controle sobre exatamente quando o processamento é concluído.
