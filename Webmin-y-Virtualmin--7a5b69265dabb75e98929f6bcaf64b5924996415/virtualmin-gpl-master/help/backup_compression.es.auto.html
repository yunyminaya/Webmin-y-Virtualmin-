Formato de compresión de respaldo
Este campo permite anular el tipo de compresión para esta copia de seguridad. Si no se establece, se utilizará el formato predeterminado global (establecido en la página de Configuración de Virtualmin).

Por defecto, Virtualmin usa la compresión gzip en sus archivos de respaldo. Sin embargo, esta opción se puede utilizar para seleccionar el formato bzip2 eficiente en espacio (pero intensivo en CPU). También puede usar copias de seguridad de formato tar sin comprimir, que pueden ser útiles si desea una copia de seguridad rápida o si está escribiendo en cinta. O el formato zip , que es más fácil de extraer en otros sistemas operativos.

Esta opción solo se usa cuando se realizan copias de seguridad de servidores virtuales en un directorio, con un archivo por dominio. Al realizar una copia de seguridad en un solo archivo, el nombre del archivo determina el formato de compresión (como domains.tar.gz o domains.tar ).
