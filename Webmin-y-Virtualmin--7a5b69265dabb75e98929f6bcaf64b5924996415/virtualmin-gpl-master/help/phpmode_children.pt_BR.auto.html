Subprocessos máximos do serviço PHP
Esta opção define o limite do número de solicitações simultâneas que serão atendidas, quando os scripts PHP para este domínio configurados forem executados via FPM, o número de processos filhos será criado quando a opção pm estiver definida como static e o número máximo de processos filhos poderia ser criado quando a opção pm estiver definida como dynamic.

Quando os scripts PHP para esta configuração de domínio são executados via FCGI, o gerenciamento de processos filhos PHP deve sempre ser desabilitado, o que encaminhará apenas uma solicitação por vez para os processos de aplicativo que ele gerou. Por padrão, e com a configuração da variável de ambiente PHP_FCGI_CHILDREN=0 , o gerenciamento de processos filhos do PHP está desabilitado.
