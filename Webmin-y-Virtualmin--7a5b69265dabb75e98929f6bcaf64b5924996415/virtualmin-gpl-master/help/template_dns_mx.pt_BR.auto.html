Nome do host para registro MX
Por padrão, o registro MX do DNS (que controla onde o email é entregue) para novos domínios será mail.${DOM}, ao qual será atribuído o mesmo endereço IP do site do servidor virtual. No entanto, você pode usar esse campo para especificar um nome de host de servidor de e-mail diferente - isso normalmente é usado para ter um registro MX consistente em todos os domínios. Certifique-se de inserir um nome de host que resolva para o seu sistema Virtualmin, ou então o e-mail não será entregue.

Você também pode usar as variáveis padrão do Virtualmin como ${DOM} para especificar um registro MX com um nome diferente no domínio que está sendo criado.
