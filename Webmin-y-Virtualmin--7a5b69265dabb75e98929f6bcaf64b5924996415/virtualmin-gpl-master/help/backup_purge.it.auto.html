Elimina i vecchi backup
Se si esegue il backup in una directory locale o bucket S3 il cui percorso si basa sul tempo di backup, questa opzione può essere utilizzata per eliminare i backup più vecchi di un certo numero di giorni. Questo può essere utile per ridurre la quantità di spazio su disco necessaria per archiviare i vecchi dati.

Tuttavia, questo è utilizzabile solo quando la directory di destinazione del backup è denominata come /backup/%d-%m-%Y , oppure il nome del file è denominato come /backup/virtualmin-%d-%m-%Y.tar.gz . Ciò è necessario affinché Virtualmin sia in grado di capire quali file e directory sono stati creati dal processo di backup.

È inoltre necessario selezionare la casella Esegui sostituzioni temporali in stile strftime sul nome del file o della directory , altrimenti la data e i codici temporali nel percorso di backup non verranno convertiti quando viene eseguito il backup.

L'età massima di backup viene immessa in giorni, ma è possibile utilizzare frazioni come 0,25 per quattro ore.
