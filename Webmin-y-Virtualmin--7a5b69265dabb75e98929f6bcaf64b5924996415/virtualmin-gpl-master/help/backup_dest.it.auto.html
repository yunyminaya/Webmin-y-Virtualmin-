Destinazione di backup
Questi campi controllano dove viene scritto il backup di Virtualmin. Puoi scegliere tra una delle tre modalità di destinazione:
Directory dei file locali
In questa modalità, il backup viene semplicemente salvato in un file sul sistema che esegue Virtualmin. Se si utilizza il formato di backup Single archive file , è necessario immettere un nome file qui, come /backup/domainname.tgz . Se si utilizza uno dei formati One file per server , è necessario immettere una directory, come /backup .
Server FTP
Questa modalità può essere utilizzata per caricare il file o i file di backup tramite il protocollo FTP su un altro server. È necessario immettere il nome del server, il login, la password e il file o la directory di destinazione.
Server SSH
È simile alla modalità FTP, ma trasferisce il backup utilizzando il protocollo SSH.
Se il formato di backup è un singolo file, allora la destinazione che specifichi qui deve essere un file, che verrà sovrascritto o creato. Se crei un file per dominio, allora la destinazione deve essere una directory che esiste già.

Se si esegue il backup su un server SSH o FTP su una porta non standard, è possibile immetterlo nel campo server separato dal nome host con due punti, ad esempio backupserver.yourdomain.com:2222 .

Se si esegue il backup su un server SSH che supporta solo l'autenticazione basata su certificato, il campo password è facoltativo. Tuttavia, l'utente root sul sistema deve avere la propria chiave pubblica SSH nel file authorized_keys dell'utente di destinazione.
