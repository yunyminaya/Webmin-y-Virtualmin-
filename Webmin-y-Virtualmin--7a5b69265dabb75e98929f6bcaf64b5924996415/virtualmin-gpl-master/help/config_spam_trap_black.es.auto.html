¿Agregar remitentes de spam enviados a trampas de spam a la lista negra?
De manera predeterminada, cuando el alias de trampa de spam está habilitado para un servidor virtual, cualquier correo electrónico que se le envíe se agregará a la base de datos de aprendizaje de SpamAssassin para el usuario que lo reenvió. Sin embargo, esta opción se puede usar para agregar también los remitentes originales de correo no deseado a las listas negras de todo el sistema o de nivel de dominio.

Esto puede ser útil si recibe una gran cantidad de spam enviado desde la misma dirección. Sin embargo, dado que los spammers a menudo varían o falsifican la dirección de origen en el correo electrónico, ponerlos en la lista negra generalmente no es muy efectivo.
