Bearbeitung der PHP-Konfiguration zulassen?
Wenn diese Option auf Ja gesetzt ist (wie standardmäßig), können Besitzer virtueller Server ihre PHP-Konfiguration bearbeiten. Sie können es jedoch auf Nein setzen , um dies zu verhindern, entweder aus Sicherheitsgründen oder um zu verhindern, dass erfahrene Administratoren falsche Änderungen vornehmen.

Dies wird implementiert, indem die Datei ~/etc/php.ini der Domain nur vom Unix- root Benutzer bearbeitet werden kann. Beachten Sie jedoch, dass dies keine narrensichere Methode ist, um Änderungen an der PHP-Konfiguration zu verhindern. Der Domänenbesitzer kann dennoch eine andere Konfigurationsdatei angeben, indem er die PHP-Wrapper-Skripte in cgi-bin bearbeitet, eine eigene Kopie von PHP ausführt oder PHP-Skripte ausführt als CGI-Programme.
