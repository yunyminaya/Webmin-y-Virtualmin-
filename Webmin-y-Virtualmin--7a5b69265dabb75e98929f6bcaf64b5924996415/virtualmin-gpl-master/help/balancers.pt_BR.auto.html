Caminhos de proxy
O proxy permite mapear um caminho de URL no site de um servidor virtual para um ou mais URLs de destino. Isso geralmente é usado para disponibilizar aplicativos Ruby on Rails atendidos por várias instâncias do Mongrel sob um caminho de URL no site do servidor. Também pode ser usado para disponibilizar outros servidores da Web, como o Tomcat, por meio de um caminho de URL no seu site.

Por exemplo, você pode mapear o caminho /radiant para os URLs http://localhost:3001 e http://localhost:3002 , que são as portas para duas instâncias do Mongrel que servem o mesmo aplicativo em execução no sistema Virtualmin.

Somente sistemas que possuem o módulo Apache mod_proxy_balancer instalado podem mapear um caminho para vários URLs. Outros que suportam apenas o módulo proxy regular do Apache podem encaminhar um caminho para apenas uma única URL.
