Limiet op aantal Mongrel gebeurtenissen
Wanneer u Ruby on Rails scripts installeert kan de server administrator selecteren hoeveel Mongrel server processen mogen starten om aanvragen af te handelen naar de pad van het script - als een Apache mod_proxy_balancer module is geinstalleerd. Om te voorkomen dat er een groot aantal Mongrel server processen worden opgestart die het systeem kunnen overbelasten kan deze optie worden gebruikt om het standaard limiet in te stellen op het aantal die domein administrators mogen configureren binnen Virtualmin.
