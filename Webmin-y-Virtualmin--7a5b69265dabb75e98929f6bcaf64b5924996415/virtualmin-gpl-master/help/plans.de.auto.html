Kontopläne
Pläne sind Sätze von Einschränkungen oder Beschränkungen, die auf virtuelle Server angewendet werden können. Jeder Plan gibt Festplattenkontingente, Beschränkungen für die Anzahl der Domänen, Benutzer und anderer Objekte sowie zulässige Funktionen und Fähigkeiten an. Beim Erstellen eines virtuellen Servers kann ein Plan ausgewählt und jederzeit auf einen vorhandenen Server angewendet werden.

Pläne beschränken die Möglichkeiten des Eigentümers einer Domain, auf die sie angewendet werden. Sie haben keine Auswirkungen auf den Master-Administrator oder auf Wiederverkäufer (die ihre eigenen separaten Limits haben).

Für jeden Plan gelten folgende Einschränkungen:

Plattenkontingente
Diese definieren die Anfangsquoten für eine Domain, sowohl insgesamt als auch für den Administrationsbenutzer.

Grenzen
Diese legen die Anzahl der Domänen, Postfächer, Aliase und Aliase fest, die der Domänenbesitzer erstellen kann. Die Grenzwerte gelten auch für alle Subserver.

Zulässige Funktionen
Diese steuern, welche Funktionen der Domänenbesitzer für vorhandene virtuelle Server aktivieren oder für neue auswählen kann, z. B. eine Website, eine DNS-Domäne oder eine E-Mail.

Fähigkeiten
Diese steuern, welche Attribute eines virtuellen Servers der Domänenbesitzer verwalten kann, z. B. E-Mail-Aliase, Datenbanken, Skripts und Sicherungen.
