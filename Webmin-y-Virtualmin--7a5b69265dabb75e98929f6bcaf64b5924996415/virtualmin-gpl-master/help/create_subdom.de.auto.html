Erstellen Sie eine virtuelle Subdomain
Mit diesem Formular können Sie eine neue virtuelle Unterdomäne erstellen, die einem vorhandenen Administrations-Login gehört. Eine Subdomäne ähnelt einem Subserver dahingehend, dass sie das Festplattenkontingent, die Bandbreitenbeschränkungen und die Beschränkungen für Benutzer, Aliase und andere Objekte mit ihrem übergeordneten Server teilt. Der Name der Unterdomäne muss jedoch unter dem Namen der übergeordneten Domäne stehen, sie verwendet ein Unterverzeichnis des Verzeichnisses public_html der übergeordneten Domäne für ihre Website und Protokolldateien werden gemeinsam genutzt.

Beim Erstellen des Servers müssen Sie mindestens den Domänennamen und eine Beschreibung eingeben. Wenn auf dem System mehrere Servervorlagen definiert sind, können Sie eine aus dem Menü auswählen. Vorlagen legen verschiedene Standardeinstellungen für den neuen Server fest, z. B. die Website-Konfiguration, DNS-Einträge usw.

Verwenden Sie die Kontrollkästchen im zweiten Teil des Abschnitts Erweiterte Optionen , um auszuwählen, welche Funktionen für den Server aktiviert sind. Subdomains verfügen im Vergleich zu Subservern oder Servern der obersten Ebene über viel weniger verfügbare Funktionen.

Im unteren Bereich des Abschnitts Erweiterte Optionen können Sie auswählen, ob der gesamte Webverkehr zu dieser neuen Site an eine andere URL weitergeleitet werden soll. Sie können auch festlegen, dass der neue virtuelle Server seine eigenen privaten IP-Adressen verwendet, die für SSL und anonymes FTP-Hosting erforderlich sind.
