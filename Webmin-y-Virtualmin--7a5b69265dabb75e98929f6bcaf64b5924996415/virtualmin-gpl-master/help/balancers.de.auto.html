Proxy-Pfade
Mit Proxying können Sie einen URL-Pfad auf der Website eines virtuellen Servers einer oder mehreren Ziel-URLs zuordnen. Dies wird normalerweise verwendet, um Ruby on Rails-Anwendungen, die von mehreren Mongrel-Instanzen bereitgestellt werden, unter einem URL-Pfad auf der Server-Website verfügbar zu machen. Es kann auch verwendet werden, um andere Webserver wie Tomcat über einen URL-Pfad auf Ihrer Website verfügbar zu machen.

Beispielsweise können Sie den Pfad /radiant den URLs http://localhost:3001 und http://localhost:3002 sind die Ports für zwei Mongrel-Instanzen, die dieselbe Anwendung bedienen, die auf Ihrem Virtualmin-System ausgeführt wird.

Nur Systeme, auf denen das Apache-Modul mod_proxy_balancer installiert ist, können einen Pfad mehreren URLs mod_proxy_balancer . Andere, die nur das reguläre Proxy-Modul von Apache unterstützen, können einen Pfad nur an eine einzelne URL weiterleiten.
