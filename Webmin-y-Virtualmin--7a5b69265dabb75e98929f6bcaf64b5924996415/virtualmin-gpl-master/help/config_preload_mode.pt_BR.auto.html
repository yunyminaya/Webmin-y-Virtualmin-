Pré-carregar bibliotecas do Virtualmin na inicialização?
Para acelerar o desempenho dos scripts CGI que compõem o Virtualmin, o Webmin pode ser configurado para carregar algumas de suas bibliotecas Perl na memória no momento da inicialização, para que elas não precisem ser lidas do disco e analisadas para cada solicitação. No entanto, isso consome mais RAM - aproximadamente 100M em um sistema Linux típico.

Essa configuração oferece mais controle sobre a quantidade de pré-carregamento realizado. As opções são:

Todas as bibliotecas
As bibliotecas para todos os módulos e plugins Webmin usados pelo Virtualmin são carregados. O processo do servidor Webmin consumirá aproximadamente 100M de RAM.
Apenas núcleo
Somente as bibliotecas que fazem parte do próprio Virtualmin são carregadas. O processo do servidor Webmin utilizará cerca de 50M de RAM.
Não
Nada é pré-carregado - em vez disso, todas as bibliotecas são carregadas sob demanda. O processo do servidor Webmin utilizará apenas 15M de RAM.
