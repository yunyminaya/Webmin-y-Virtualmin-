Transferir servidor virtual
Esta página se puede usar para enviar un servidor virtual (y cualquier sub-servidor o alias) a otro sistema que ejecute Virtualmin. El proceso de transferencia usando la funcionalidad de respaldo y restauración de Virtualmin, e incluirá todo el contenido del directorio de inicio, bases de datos, registros DNS y otras configuraciones.

El nombre de host o la dirección IP del sistema de destino deben ingresarse en el campo Nombre de host del sistema de destino , y la contraseña de Root para el campo del sistema de destino debe completarse a menos que tenga una configuración de autenticación SSH sin contraseña.

Por defecto, el dominio simplemente se copiará al nuevo sistema. Sin embargo, el campo Acción después de la transferencia se puede utilizar para moverlo, eliminando el servidor del sistema de origen. Esto es necesario si tanto el origen como el destino comparten un servidor DNS agrupado o son servidores de nombres secundarios entre sí.
