Mail Rate Limiting
Auf dieser Seite können Sie steuern, mit welcher Geschwindigkeit der Mailserver Ihres Systems neue Nachrichten akzeptiert. Standardmäßig kann jedes Skript auf dem System oder Remote-Absender mit einem SMTP-Login E-Mails ohne Ratenbeschränkungen senden. Dies kann ein Problem sein, wenn ein Login oder ein Skript von einem Spammer entführt wird. Mit dieser Funktion können Sie ein angemessenes Limit festlegen, z. B. 1000 Nachrichten pro Stunde im gesamten System.

Zusätzlich zu einem globalen Limit können Sie auch Domain-Raten pro Absender festlegen. Diese überschreiben jedes globale Limit und können verwendet werden, um einer bestimmten vertrauenswürdigen Domäne ein höheres oder niedrigeres Limit als das für andere geltende zu gewähren.
