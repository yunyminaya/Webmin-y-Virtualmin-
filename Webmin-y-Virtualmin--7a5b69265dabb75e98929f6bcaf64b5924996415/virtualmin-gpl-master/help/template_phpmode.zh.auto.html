默认 PHP 执行模式
默认的 PHP 脚本执行模式定义了如何在新的虚拟服务器中处理 PHP 代码。

禁用模式意味着服务器不会执行 PHP 脚本，而是将其视为纯文本文件。当不需要 PHP 处理或将 PHP 代码作为可下载文件提供时，可以使用此模式。

FPM是首选模式，可提供高效率和自适应进程管理等高级功能。它作为独立的 FastCGI 服务器运行，由于其强大的性能，非常适合高流量站点。

FCGI对于无法使用 FPM 的服务器非常有用。它还支持每个虚拟主机的配置，尽管在可扩展性和管理功能方面它通常比不上 FPM。

CGI模式通过 CGI 协议将每个 PHP 脚本作为单独的进程运行。此模式的效率低于 FPM 或 FCGI，因为需要为每个请求创建新进程。由于其性能限制，它在现代设置中很少使用。

mod_php将 PHP 直接集成到 Apache Web 服务器中，允许 PHP 脚本在服务器自己的进程中执行。但是，由于安全和隔离问题，加上停止支持和更新，它现在已经过时了。
