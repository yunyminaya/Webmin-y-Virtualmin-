Creazione in batch di server
Questa pagina consente di creare più server virtuali contemporaneamente da un file di testo. Questo file deve elencare un server per riga e ogni riga deve avere il seguente formato:

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

I significati dei campi sono:

dominio
Il nome di dominio per il server virtuale da creare, ad esempio foo.com .
descrizione
Una descrizione leggibile dall'uomo, come il sito Web di Joe .
parola d'ordine
La password per l'accesso di amministrazione del server. Questo è necessario solo per i server di appartenenza.
nome utente
Il nome utente di amministrazione per il server. Se questo viene lasciato vuoto, viene automaticamente selezionato un nome utente.
genitore-dominio
Il nome di dominio del server padre in cui creare questo server. Se questo viene lasciato vuoto, il nuovo server viene creato come genitore stesso.
indirizzo IP
Per i server virtuali che non necessitano del proprio IP privato, questo campo può essere lasciato vuoto. Altrimenti, è necessario immettere un indirizzo non utilizzato valido oppure la parola allocate per fare in modo che Virtualmin ne scelga uno (se configurato).
alias dominio
Il nome di dominio del server virtuale per cui dovrebbe diventare un alias. Se lasciato fuori, il nuovo server sarà di primo livello o un sub-server.

Le quote, la larghezza di banda e altri limiti per i server creati vengono impostati in base al modello selezionato in questo modulo. Le funzionalità per i server creati sono determinate dalle caselle di controllo delle funzionalità, proprio come nella normale pagina di creazione a server singolo.

Se il tuo sistema utilizza gli account rivenditore, puoi scegliere di creare tutti i server virtuali di proprietà di un rivenditore. Questa opzione è disponibile solo per l'amministratore principale.
