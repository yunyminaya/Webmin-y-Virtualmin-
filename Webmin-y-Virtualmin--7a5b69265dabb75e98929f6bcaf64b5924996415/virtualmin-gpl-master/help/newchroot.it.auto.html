Limitazioni della directory FTP
Questa pagina consente di configurare quali directory possono vedere i proprietari dei server virtuali e gli utenti con accesso FTP quando accedono tramite FTP. Per impostazione predefinita, i client FTP possono accedere all'intero filesystem, comprese le directory come /etc e /home . Tuttavia, ciò potrebbe fornire informazioni che preferiresti che gli utenti non potessero vedere.

Ogni riga nella tabella in questa pagina definisce il server o i server virtuali corrispondenti e la directory a cui sono limitati. Nella colonna Applica a server e server secondari , puoi scegliere tutti i server virtuali o un singolo dominio (che includerà automaticamente tutti i relativi server secondari). Se viene scelta quest'ultima opzione e la casella Tutti tranne server è selezionata, la restrizione avrà effetto su server diversi da quello scelto.

La colonna Limita alla directory determina dove sono limitati i server virtuali corrispondenti quando si accede tramite FTP. L'opzione più comunemente usata sono le home directory degli utenti , che limitano il proprietario del dominio alla home del server virtuale e tutti gli utenti FTP alle proprie sottodirectory, in genere nelle homes .

In un'installazione standard di Virtualmin, la home directory di ogni dominio ha 750 autorizzazioni, il che impedisce agli utenti di altri domini di accedervi, indipendentemente dalle restrizioni della directory FTP. Quindi bloccare gli utenti FTP in questa pagina non aggiunge molto alla sicurezza del tuo sistema.

Inoltre, le restrizioni in questa pagina non si applicano agli script PHP, ai programmi CGI o ai proprietari di server virtuali che accedono tramite SSH. Al momento, non è possibile bloccarli, anche se le autorizzazioni della directory principale sono ancora valide.
