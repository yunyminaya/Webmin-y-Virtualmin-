¿Configurar el trabajo Webalizer Cron para cada servidor virtual?
Cuando esta opción se establece en Sí (como está por defecto), Virtualmin creará un trabajo Cron separado para cada servidor virtual que tenga Webalizer habilitado, a fin de generar un informe de sus registros diariamente.

Sin embargo, en un servidor con una gran cantidad de dominios, puede cambiar esta opción a No y programar manualmente el comando /usr/libexec/webmin/virtual-server/run-all-webalizer.pl usando Cron. Esto asegurará que Webalizer se ejecute para cada dominio por turnos en lugar de potencialmente tener múltiples dominios procesados al mismo tiempo, y le brinda más control sobre exactamente cuándo se realiza el procesamiento.
