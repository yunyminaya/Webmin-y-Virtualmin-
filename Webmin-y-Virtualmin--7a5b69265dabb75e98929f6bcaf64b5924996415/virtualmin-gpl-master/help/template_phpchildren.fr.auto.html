Sous-processus maximum du service PHP
Cette option définit la limite du nombre de requêtes simultanées qui seront servies, lorsque les scripts PHP pour ce domaine sont configurés pour s'exécuter via FPM, le nombre de processus enfants seront créés lorsque l'option pm est définie sur static et le nombre maximum de processus enfants. pourrait potentiellement être créé lorsque l'option pm est définie sur dynamic.

Lorsque les scripts PHP pour cette configuration de domaine sont exécutés via FCGI, la gestion des processus enfants PHP doit toujours être désactivée, ce qui n'acheminera qu'une seule requête à la fois vers les processus d'application qu'elle a générés. Par défaut, et avec le paramètre de variable d'environnement PHP_FCGI_CHILDREN=0 , la gestion des processus enfants PHP est désactivée.
