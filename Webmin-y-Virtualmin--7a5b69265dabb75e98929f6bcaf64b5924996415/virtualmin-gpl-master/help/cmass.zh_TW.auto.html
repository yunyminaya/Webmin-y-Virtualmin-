批量創建服務器
該頁面允許您從一個文本文件一次創建多個虛擬服務器。該文件必須每行列出一台服務器，並且每行必須具有以下格式：

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

這些字段的含義是：

域
要創建的虛擬服務器的域名，例如foo.com 。
描述
易於理解的描述，例如Joe's Website 。
密碼
服務器管理登錄名的密碼。僅父服務器才需要。
用戶名
服務器的管理用戶名。如果將此留空，則會自動選擇一個用戶名。
父域
在其下創建此服務器的父服務器的域名。如果將其保留為空，則將新服務器本身創建為父服務器。
IP地址
對於不需要自己的專用IP的虛擬服務器，此字段可以保留為空白。否則，您必須輸入一個有效的未使用地址，或使用“ allocate ”一詞使Virtualmin為您選擇一個（如果已配置）。
別名域
應該成為其別名的虛擬服務器的域名。如果省略，則新服務器將是頂級服務器或子服務器。

已創建服務器的配額，帶寬和其他限制是根據您在此表單上選擇的模板設置的。就像正常的單服務器創建頁面一樣，創建的服務器的功能由功能複選框確定。

如果您的系統使用代理商帳戶，則可以選擇在代理商的所有權下創建所有虛擬服務器。此選項僅對主管理員可用。
