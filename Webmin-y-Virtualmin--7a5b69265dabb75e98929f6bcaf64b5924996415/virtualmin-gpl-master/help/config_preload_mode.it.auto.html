Precaricare le librerie Virtualmin all'avvio?
Al fine di accelerare le prestazioni degli script CGI che compongono Virtualmin, Webmin può essere configurato per caricare alcune delle loro librerie Perl in memoria al momento dell'avvio in modo che non debbano essere lette dal disco e analizzate per ogni richiesta. Tuttavia, questo utilizza più RAM, circa 100 milioni su un tipico sistema Linux.

Questa impostazione ti offre un maggiore controllo sulla quantità di pre-caricamento. Le opzioni sono:

Tutte le biblioteche
Vengono caricate le librerie per tutti i moduli e plugin Webmin utilizzati da Virtualmin. Il processo del server Webmin consumerà circa 100 milioni di RAM.
Solo core
Vengono caricate solo le librerie che fanno parte di Virtualmin stesso. Il processo del server Webmin utilizzerà circa 50 MB di RAM.
No
Nulla è precaricato, ma tutte le librerie sono caricate su richiesta. Il processo del server Webmin utilizzerà solo 15 MB di RAM.
