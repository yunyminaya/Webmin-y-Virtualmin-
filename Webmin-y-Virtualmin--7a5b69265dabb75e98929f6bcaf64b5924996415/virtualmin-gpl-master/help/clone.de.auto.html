Virtuellen Server klonen
Auf dieser Seite können Sie einen vorhandenen virtuellen Server mit einem neuen Domänennamen duplizieren. Alle mit der ursprünglichen Domäne verknüpften Webinhalte, DNS-Einträge, Postfächer, E-Mail-Aliase, Datenbanken und andere Einstellungen werden, soweit möglich, dupliziert. Die Webinhalte werden jedoch in keiner Weise geändert. Wenn der virtuelle Server also eine Webanwendung enthält, die so konfiguriert wurde, dass sie die Datenbanken oder das Stammverzeichnis des alten Servers verwendet, muss diese nach dem Klonen manuell neu konfiguriert werden.

Beim Duplizieren eines virtuellen Top-Level-Servers müssen Sie außerdem einen neuen Benutzernamen und ein neues Passwort eingeben, um Eigentümer der neuen Domäne zu werden. Wenn der Server eine private IP-Adresse hat, wird dem Klon automatisch eine neue IP zugewiesen.

Der Klonvorgang kopiert alle mit der ursprünglichen Domäne verknüpften Datenbanken unter neuen Namen und dupliziert deren Inhalt. Allerdings werden dabei möglicherweise nicht alle in der ursprünglichen Domäne installierten Webanwendungen aktualisiert, die für die Verwendung der ursprünglichen Datenbanken konfiguriert sind. Dies muss manuell erfolgen.
