Criar servidores em lote
Esta página permite criar vários servidores virtuais ao mesmo tempo a partir de um arquivo de texto. Este arquivo deve listar um servidor por linha e cada linha deve ter o seguinte formato:

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

Os significados dos campos são:

domínio
O nome do domínio a ser criado pelo servidor virtual, como foo.com .
descrição
Uma descrição legível por humanos, como o site de Joe .
senha
A senha para o login de administração do servidor. Isso é necessário apenas para servidores-pai.
nome do usuário
O nome de usuário de administração do servidor. Se isso for deixado em branco, um nome de usuário é selecionado automaticamente.
domínio pai
O nome de domínio do servidor pai no qual o servidor será criado. Se isso for deixado vazio, o novo servidor será criado como um pai.
endereço de IP
Para servidores virtuais que não precisam de seu próprio IP privado, esse campo pode ser deixado em branco. Caso contrário, você deve inserir um endereço não utilizado válido ou a palavra allocate para que o Virtualmin selecione um para você (se configurado).
alias-domain
O nome de domínio do servidor virtual para o qual este deve se tornar um alias. Se deixado de fora, o novo servidor será de nível superior ou um sub-servidor.

Cotas, largura de banda e outros limites para os servidores criados são definidos com base no modelo que você seleciona neste formulário. Os recursos para os servidores criados são determinados pelas caixas de seleção dos recursos, assim como na página normal de criação de um único servidor.

Se o seu sistema usa contas de revendedor, você pode optar por ter todos os servidores virtuais criados sob a propriedade de um revendedor. Esta opção está disponível apenas para o administrador principal.
