Programa cliente SpamAssassin
Esta opción determina si el filtrado de spam se realiza utilizando el programa spamassassin independiente o el cliente ligero spamc . En este último caso, el sistema debe ejecutar el spamd servidor, o no se realizará ninguna clasificación de spam.

El uso de spamc suele ser más rápido y consume menos memoria y CPU. Sin embargo, eliminará la capacidad de tener una configuración diferente de SpamAssassin para cada servidor virtual, ya que el servidor de spamd solo puede leer un solo conjunto de archivos de configuración.
