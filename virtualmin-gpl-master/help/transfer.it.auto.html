Trasferisci server virtuale
Questa pagina può essere utilizzata per inviare un server virtuale (e tutti i server secondari o alias) a un altro sistema che esegue Virtualmin. Il processo di trasferimento utilizza la funzionalità di backup e ripristino di Virtualmin e includerà tutto il contenuto della directory home, i database, i record DNS e altre configurazioni.

Il nome host o l'indirizzo IP del sistema di destinazione deve essere inserito nel campo Nome host del sistema di destinazione e il campo Password di root per il sistema di destinazione deve essere compilato a meno che non si abbia un'impostazione di autenticazione SSH senza password.

Per impostazione predefinita, il dominio verrà semplicemente copiato nel nuovo sistema. Tuttavia, è possibile utilizzare il campo Azione dopo trasferimento per spostarlo, eliminando il server dal sistema di origine. Ciò è necessario se l'origine e la destinazione condividono un server DNS cluster o sono server dei nomi secondari l'uno per l'altro.
