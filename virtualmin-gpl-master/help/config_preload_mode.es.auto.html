¿Precargar las bibliotecas de Virtualmin al inicio?
Para acelerar el rendimiento de los scripts CGI que componen Virtualmin, Webmin se puede configurar para cargar algunas de sus bibliotecas Perl en la memoria en el momento del inicio para que no tengan que leerse del disco y analizarse para cada solicitud. Sin embargo, esto usa más RAM, aproximadamente 100M en un sistema Linux típico.

Esta configuración le da más control sobre la cantidad de carga previa que se realiza. Las opciones son:

Todas las bibliotecas
Se cargan las bibliotecas para todos los módulos y complementos de Webmin utilizados por Virtualmin. El proceso del servidor Webmin consumirá aproximadamente 100M de RAM.
Solo núcleo
Solo se cargan las bibliotecas que forman parte de Virtualmin. El proceso del servidor Webmin usará alrededor de 50M de RAM.
No
Nada está precargado; en cambio, todas las bibliotecas se cargan a pedido. El proceso del servidor Webmin usará solo 15M de RAM.
