Bewerken van de PHP configuratie toestaan?
Wanneer deze optie is ingesteld op Ja (zoals het standaard is) kunnen virtuele server eigenaars hun PHP configuratie bewerken. U kunt dit ook op Nee zetten voor veiligheid's risico's of om te voorkomen dat onervaren administrators verkeerde wijzigingen maken.

Dit word gedaan door de domein ~/etc/php.ini file root alleen voor de Unix gebruiker bewerkbaar te maken. Wees ervan bewust dat dit niet een honderd procent veilige methoden is om wijzigingen aan de PHP configuratie te voorkomen - de domein eigenaar kan nog altijd een andere configuratie file specificeren door de PHP wrapper scripts in de cgi-bin te bewerken, om vervolgens hun eigen kopie van PHP te laten werken of PHP scripts te laten werken als CGI programma's.
