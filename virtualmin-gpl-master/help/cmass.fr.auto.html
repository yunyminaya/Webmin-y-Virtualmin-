Créer des serveurs par lots
Cette page vous permet de créer plusieurs serveurs virtuels à la fois à partir d'un fichier texte. Ce fichier doit répertorier un serveur par ligne et chaque ligne doit avoir le format suivant:

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

Les significations des champs sont:

domaine
Le nom de domaine pour le serveur virtuel à créer, tel que foo.com .
la description
Une description lisible par l'homme, comme le site Web de Joe .
mot de passe
Mot de passe pour la connexion d'administration du serveur. Cela n'est nécessaire que pour les serveurs parents.
Nom d'utilisateur
Nom d'utilisateur d'administration du serveur. Si ce champ n'est pas renseigné, un nom d'utilisateur est sélectionné automatiquement.
domaine-parent
Le nom de domaine du serveur parent sous lequel créer ce serveur. S'il est laissé vide, le nouveau serveur est créé en tant que parent lui-même.
adresse IP
Pour les serveurs virtuels qui n'ont pas besoin de leur propre adresse IP privée, ce champ peut être laissé vide. Sinon, vous devez saisir une adresse inutilisée valide ou le mot allocate pour que Virtualmin en sélectionne une pour vous (si elle est configurée).
domaine-alias
Nom de domaine du serveur virtuel pour lequel cela doit devenir un alias. S'il est omis, le nouveau serveur sera soit de niveau supérieur, soit un sous-serveur.

Les quotas, la bande passante et d'autres limites pour les serveurs créés sont définis en fonction du modèle que vous sélectionnez sur ce formulaire. Les fonctionnalités des serveurs créés sont déterminées par les cases à cocher des fonctionnalités, tout comme sur la page de création de serveur unique normale.

Si votre système utilise des comptes revendeur, vous pouvez choisir de créer tous les serveurs virtuels sous la propriété d'un revendeur. Cette option n'est disponible que pour l'administrateur principal.
