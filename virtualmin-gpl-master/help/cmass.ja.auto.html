サーバーのバッチ作成
このページでは、テキストファイルから一度に複数の仮想サーバーを作成できます。このファイルには1行に1つのサーバーがリストされている必要があり、各行の形式は次のとおりです。

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

フィールドの意味は次のとおりです。

ドメイン
作成する仮想サーバーのドメイン名（ foo.comなど）。
解説
Joe's Websiteなど、人間が読める形式の説明。
パスワード
サーバーの管理ログインのパスワード。これは、親サーバーでのみ必要です。
ユーザー名
サーバーの管理ユーザー名。空白のままにすると、ユーザー名が自動的に選択されます。
親ドメイン
このサーバーを作成する親サーバーのドメイン名。これを空のままにすると、新しいサーバーが親自体として作成されます。
IPアドレス
独自のプライベートIPを必要としない仮想サーバーの場合、このフィールドは空白のままにすることができます。それ以外の場合は、有効な未使用のアドレスを入力するか、「 allocateという単語でVirtualminが1つを選択するようにします（構成されている場合）。
エイリアスドメイン
これがエイリアスになる仮想サーバーのドメイン名。省略した場合、新しいサーバーはトップレベルまたはサブサーバーになります。

作成されたサーバーのクォータ、帯域幅、その他の制限は、このフォームで選択したテンプレートに基づいて設定されます。作成されたサーバーの機能は、通常の単一サーバー作成ページと同様に、機能チェックボックスによって決定されます。

システムがリセラーアカウントを使用している場合、リセラーの所有権の下にすべての仮想サーバーを作成するように選択できます。このオプションは、マスター管理者のみが使用できます。
