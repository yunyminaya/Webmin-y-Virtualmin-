Créer un sous-domaine virtuel
Ce formulaire vous permet de créer un nouveau sous-domaine virtuel, qui appartient à une connexion d'administration existante. Un sous-domaine est similaire à un sous-serveur, en ce sens qu'il partage le quota de disque, les limites de bande passante et les limites d'utilisateurs, d'alias et d'autres objets avec son parent. Cependant, le nom du sous-domaine doit être sous le nom du domaine parent, il utilise un sous-répertoire du répertoire public_html du parent pour son site Web et les fichiers journaux sont partagés.

Lors de la création du serveur, vous devez saisir au moins le nom de domaine et une description. Si plusieurs modèles de serveur sont définis sur le système, vous pouvez en sélectionner un dans le menu - les modèles déterminent diverses valeurs par défaut pour le nouveau serveur, telles que la configuration du site Web, les enregistrements DNS, etc.

Pour sélectionner les fonctionnalités qui sont activées pour le serveur, utilisez les cases à cocher dans la deuxième partie de la section Options avancées . Les sous-domaines ont un ensemble de fonctionnalités disponibles beaucoup plus petit que les sous-serveurs ou les serveurs de niveau supérieur.

Au bas de la section Options avancées, vous pouvez choisir de transférer tout le trafic Web vers ce nouveau site vers une autre URL. Vous pouvez également choisir que le nouveau serveur virtuel utilise ses propres adresses IP privées, ce qui est requis pour SSL et l'hébergement FTP anonyme.
