Rediriger les sous-répertoires vers
Cette option contrôle la manière dont les chemins sous le chemin de l'URL source sont redirigés. Par exemple, si le chemin source est /foo et la destination est http://foo.com la signification des options est :

  • Même sous-répertoire sous l'URL de destination
    Une requête vers /foo/bar sera redirigée vers http://foo.com/bar

  • Uniquement l'URL de destination
    Une requête vers /foo/bar sera simplement redirigée vers http://foo.com

  • Ignorer les sous-répertoires
    Une requête vers /foo/bar ne sera pas du tout redirigée, seule une requête vers /foo le sera.
