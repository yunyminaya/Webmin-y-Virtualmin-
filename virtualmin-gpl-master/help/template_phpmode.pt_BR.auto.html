Modo de execução padrão do PHP
O modo de execução padrão do script PHP define como o código PHP será processado em novos servidores virtuais.

O modo desativado significa que os scripts PHP não são executados pelo servidor, mas são tratados como arquivos de texto simples. Este modo pode ser usado quando o processamento de PHP não é necessário ou ao servir código PHP como arquivos para download.

FPM é o modo preferido, oferecendo alta eficiência e recursos avançados como gerenciamento adaptativo de processos. Ele opera como um servidor FastCGI independente, ideal para sites de alto tráfego devido às suas robustas capacidades de desempenho.

FCGI é útil para servidores onde o FPM não está disponível. Ele também oferece suporte a configurações por host virtual, embora geralmente não corresponda ao FPM em termos de escalabilidade e recursos de gerenciamento.

O modo CGI executa cada script PHP como um processo separado através do protocolo CGI. Este modo é menos eficiente que FPM ou FCGI devido à sobrecarga de criação de um novo processo para cada solicitação. Raramente é usado em configurações modernas devido às suas limitações de desempenho.

mod_php integra PHP diretamente no servidor web Apache, permitindo que scripts PHP sejam executados dentro do próprio processo do servidor. No entanto, devido a questões de segurança e isolamento, além da descontinuação do suporte e das atualizações, agora está obsoleto.
