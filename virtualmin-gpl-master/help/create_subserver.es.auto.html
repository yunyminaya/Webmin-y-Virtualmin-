Crear sub-servidor virtual
Este formulario le permite crear un nuevo subservidor virtual, que es propiedad de un inicio de sesión de administración existente. Un subservidor también comparte la cuota de disco, los límites de ancho de banda y los límites de usuarios, alias y otros objetos con su padre.

Al crear el servidor, debe ingresar al menos el nombre de dominio y una descripción. Si se definen varias plantillas de servidor en el sistema, puede seleccionar una del menú: las plantillas determinan varios valores predeterminados para el nuevo servidor, como la configuración del sitio web, los registros DNS, etc.

En la sección Opciones avanzadas , puede seleccionar un nombre de base de datos inicial diferente, en lugar de utilizar el predeterminado que se deriva del nombre de dominio. Los nombres de usuario y grupo no se pueden cambiar, ya que siempre se heredan del servidor principal.

Para seleccionar qué funciones están habilitadas para el servidor, use las casillas de verificación en la segunda parte de la sección Opciones avanzadas . Algunas funciones dependen de otras y no se pueden seleccionar por sí mismas; por ejemplo, los informes de Webalizer sobre las estadísticas del sitio web dependen de tener un sitio web.

En la parte inferior de la sección Opciones avanzadas , puede seleccionar que se reenvíe todo el tráfico web a este nuevo sitio a otra URL y que se reenvíe todo el correo electrónico. También puede seleccionar que el nuevo servidor virtual use sus propias direcciones IP privadas, que se requieren para el alojamiento SSL y FTP anónimo.
