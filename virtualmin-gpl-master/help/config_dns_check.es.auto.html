¿Verifique resolv.conf para este sistema?
Cuando se establece en Sí (como lo está por defecto), Virtualmin verificará el archivo de configuración del cliente DNS del sistema /etc/resolv.conf para asegurarse de que esté configurado para usar el servidor DNS local para la resolución de nombres. Esto es necesario, ya que sin él y los dominios DNS que cree pueden no ser resueltos por el sistema que ejecuta Virtualmin hasta que estén completamente registrados por un registrador DNS.

Si está seguro de que esto no es necesario (tal vez porque tiene una configuración de servidor DNS secundario automático), esto puede cambiarse a No para evitar errores de comprobación de configuración espurios.
