Sauvegarder MySQL en une seule transaction ?
Lorsque cette option est définie sur Yes , les sauvegardes MySQL seront effectuées en une seule transaction pour assurer la cohérence. Cependant, cela peut consommer plus de RAM lors de la sauvegarde d'une base de données volumineuse et changeante. Il peut également échouer avec l'erreur Couldn't execute 'FLUSH TABLES WITH READ LOCK' sur certaines versions de MySQL ou Amazon RDS. Dans ce cas, vous devez le remplacer par Non .
