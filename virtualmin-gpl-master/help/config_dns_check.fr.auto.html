Vérifiez resolv.conf pour ce système?
Lorsque ce paramètre est défini sur Oui (comme c'est le cas par défaut), Virtualmin vérifie le fichier de configuration du client DNS du système /etc/resolv.conf pour s'assurer qu'il est configuré pour utiliser le serveur DNS local pour la résolution de noms. Cela est nécessaire, car sans lui, les domaines DNS que vous créez peuvent ne pas être résolus par le système exécutant Virtualmin jusqu'à ce qu'ils soient entièrement enregistrés par un registraire DNS.

Si vous êtes certain que cela n'est pas nécessaire (peut-être parce que vous avez une configuration automatique de serveur DNS secondaire), cela peut être changé en Non pour éviter les erreurs de vérification de configuration parasites.
