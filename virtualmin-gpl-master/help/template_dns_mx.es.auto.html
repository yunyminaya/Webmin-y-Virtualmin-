Nombre de host para registro MX
De forma predeterminada, el registro DNS MX (que controla dónde se entrega el correo electrónico) para los nuevos dominios será mail.${DOM}, al que se le asignará la misma dirección IP que el sitio web del servidor virtual. Sin embargo, puede usar este campo para especificar un nombre de host de servidor de correo diferente; esto generalmente se usa para tener un registro MX consistente en todos los dominios. Sin embargo, asegúrese de ingresar un nombre de host que se resuelva en su sistema Virtualmin, o de lo contrario no se entregará el correo electrónico.

También puede usar las variables estándar de Virtualmin como ${DOM} para especificar un registro MX con un nombre diferente en el dominio que se está creando.
