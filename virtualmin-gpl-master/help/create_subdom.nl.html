Virtueel Sub-Domein aanmaken
Met dit formulier kunt u een nieuw virtueel sub-domein aanmaken die een nieuwe bijbehorende administratieve login heeft. Een sub-domein is vergelijkbaar met een sub-server, waarin het harddisk quota deelt, bandbreedte limieten en limieten op gebruikers, aliassen en andere objecten met zijn hoofd server. Echter de sub-domein naam moet onder de hoofd domein, het gebruikt een sub-directory van zijn hoofd server zijn public_html directory voor zijn website en ook de log files worden gedeeld.

Wanneer u een server aanmaakt moet u op zijn minst een domein naam opgeven en een omschrijving. Als er meerdere server templates zijn gedefinieerd op het systeem kunt u uit het menu selecteren wat u wenst - template's bepalen verschillende standaard instellingen voor de nieuwe server, zoals de website configuratie, DNS records enzovoort.

Om te selecteren welke functies er aan moeten staan voor de server gebruikt u de selectievakjes in het tweede gedeelte van de Geavanceerde opties sectie. Sub-domeinen hebben veel minder beschikbare functies in vergelijking met sub-servers of top-niveau servers.

Aan de onderkant van de Geavanceerde opties sectie kunt u selecteren om al het web verkeer naar deze nieuwe website moet worden doorgestuurd naar een andere URL. U kunt ook selecteren dat de nieuwe virtuele server zijn eigen IP adres moet gebruiken die nodig is voor SSL en anonieme FTP hosting.
