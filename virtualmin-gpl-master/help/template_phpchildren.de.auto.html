Maximale Anzahl von Unterprozessen des PHP-Dienstes
Diese Option legt die Begrenzung der Anzahl gleichzeitiger Anfragen fest, die bedient werden, wenn PHP-Skripte für diese Domäne so eingerichtet werden, dass sie über FPM ausgeführt werden. Sie enthält die Anzahl der untergeordneten Prozesse, die erstellt werden, wenn pm Option auf „ static eingestellt ist, und die maximale Anzahl untergeordneter Prozesse, die potenziell erstellt werden können, wenn pm Option auf „ dynamic.

Wenn PHP-Skripte für diese Domäne so eingerichtet werden, dass sie über FCGI ausgeführt werden, sollte die Verwaltung untergeordneter PHP-Prozesse immer deaktiviert sein, damit jeweils nur eine Anfrage an die von ihr erzeugten Anwendungsprozesse weitergeleitet wird. Standardmäßig und mit der Umgebungsvariableneinstellung PHP_FCGI_CHILDREN=0 ist die Verwaltung untergeordneter PHP-Prozesse deaktiviert.
