Permitir edição da configuração do PHP?
Quando esta opção é definida como Sim (como é por padrão), os proprietários do servidor virtual têm permissão para editar sua configuração do PHP. No entanto, você pode configurá-lo como Não para evitar isso, por motivos de segurança ou para impedir que administradores experientes façam alterações incorretas.

Isso é implementado tornando o arquivo ~/etc/php.ini do domínio editável apenas pelo usuário Unix root . Esteja ciente de que este não é um método infalível de impedir alterações na configuração do PHP - o proprietário do domínio ainda pode especificar um arquivo de configuração diferente editando os scripts do wrapper PHP no cgi-bin , executando sua própria cópia do PHP ou executando scripts do PHP como programas CGI.
