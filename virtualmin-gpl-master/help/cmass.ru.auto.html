Пакетное создание серверов
Эта страница позволяет вам создать несколько виртуальных серверов одновременно из текстового файла. Этот файл должен содержать один сервер в каждой строке, и каждая строка должна иметь следующий формат:

domain : description : password :[ username ]:[ parent-domain ]:[ ip-address ]:[ alias-domain ]

Значения полей:

домен
Доменное имя для создаваемого виртуального сервера, такого как foo.com .
описание
Удобочитаемое описание, такое как веб-сайт Джо .
пароль
Пароль для входа администратора сервера. Это необходимо только для родительских серверов.
имя пользователя
Имя администратора для сервера. Если оставить это поле пустым, имя пользователя будет выбрано автоматически.
родитель-домен
Доменное имя родительского сервера для создания этого сервера. Если оставить это поле пустым, новый сервер будет создан как родитель.
айпи адрес
Для виртуальных серверов, которым не нужен собственный частный IP-адрес, это поле можно оставить пустым. В противном случае, вы должны либо ввести правильный Неиспользуемый адрес, или слово allocate иметь Virtualmin выбрать один для вас (если настроено).
псевдоним домена
Доменное имя виртуального сервера, для которого это должно стать псевдонимом. Если не указан, новый сервер будет либо верхнего уровня, либо подсервер.

Квоты, пропускная способность и другие ограничения для создаваемых серверов устанавливаются на основе шаблона, выбранного в этой форме. Функции для созданных серверов определяются флажками функций, как на обычной странице создания одного сервера.

Если в вашей системе используются учетные записи посредников, вы можете выбрать, чтобы все виртуальные серверы создавались под управлением посредника. Эта опция доступна только для главного администратора.
