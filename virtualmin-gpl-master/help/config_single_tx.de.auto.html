Backup von MySQL in einer einzigen Transaktion?
Wenn diese Option auf Ja gesetzt ist, werden MySQL-Sicherungen in einer einzigen Transaktion durchgeführt, um Konsistenz zu gewährleisten. Dies kann jedoch mehr RAM verbrauchen, wenn eine große und sich ändernde Datenbank gesichert wird. Bei einigen MySQL-Versionen oder Amazon RDS kann es auch mit dem Fehler „FLUSH TABLES WITH READ LOCK“ konnte nicht ausgeführt werden . In diesem Fall sollten Sie dies stattdessen auf Nein ändern.
