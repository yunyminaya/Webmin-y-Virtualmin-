Modo de ejecución de script Ruby
Esta opción determina si los scripts Ruby se ejecutan como el usuario del servidor web Apache (normalmente httpd o apache ) o como el usuario Unix propietario del servidor virtual. El primero es más rápido porque significa que no es necesario iniciar procesos adicionales para ejecutar el código Ruby, pero es menos seguro ya que los scripts leen y escriben archivos como un usuario distinto del propietario del dominio.
