Autoriser l'édition de la configuration PHP?
Lorsque cette option est définie sur Oui (comme c'est le cas par défaut), les propriétaires de serveurs virtuels sont autorisés à modifier leur configuration PHP. Cependant, vous pouvez le définir sur Non pour éviter cela, soit pour des raisons de sécurité, soit pour empêcher les administrateurs inexpérimentés d'apporter des modifications incorrectes.

Ceci est implémenté en rendant le fichier ~/etc/php.ini du domaine modifiable uniquement par l'utilisateur Unix root . Sachez que ce n'est pas une méthode infaillible pour empêcher les modifications de la configuration PHP - le propriétaire du domaine pourrait toujours spécifier un fichier de configuration différent en modifiant les scripts PHP wrapper dans cgi-bin , en exécutant leur propre copie de PHP ou en exécutant des scripts PHP comme programmes CGI.
