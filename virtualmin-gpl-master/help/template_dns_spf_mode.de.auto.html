SPF-DNS-Eintrag hinzufügen?
SPF ( Sender Policy Framework ) ist ein Mittel, um Spam zu reduzieren, indem einige Einschränkungen für das Senden von E-Mails durch Hosts durchgesetzt werden, die angeblich von welcher Domain stammen. Da die überwiegende Mehrheit der Spam-Mails von gefälschten Domain-Namen stammt, wird von vielen angenommen, dass das Verhindern gefälschter Domain-Namen in Spam das Gesamtvolumen an Spam verringert. Wenn aktiviert, erstellt Virtualmin automatisch einen SPF-Eintrag auf Ihrem DNS-Server. Dieser Datensatz kann entweder bestimmte Hosts enthalten, die E-Mails unter Verwendung des betreffenden Domänennamens senden dürfen (wobei der Server selbst der erste Host dieser Art ist, aber eine beliebige Anzahl von Hosts aufgelistet werden kann), oder einen generischen Sammelbegriff, dh einen beliebigen Der Host kann E-Mails senden, die angeblich von dieser Domain stammen. Die zweite Option macht den Zweck von SPF effektiv ungültig, wird jedoch bereitgestellt, um Benutzern das Senden von E-Mails von ihrer eigenen Domäne über verschiedene ISP-Mailserver zu erleichtern.
