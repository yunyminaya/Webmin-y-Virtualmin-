Les champs personnalisés
Les champs personnalisés permettent à l'administrateur d'inclure des données supplémentaires sur les comptes de domaine. Les champs seront affichés lors de la modification du domaine. Ils peuvent être utilisés pour à peu près n'importe quel type de données. Par exemple, il peut être utile d'avoir des informations de contact supplémentaires attachées au compte de domaine, ou un champ de notes utilisé pour expliquer pourquoi un domaine est désactivé ou a une fonctionnalité ou une limite supplémentaire au-delà d'autres domaines créés avec le même modèle. Si vous développez des outils pour interagir avec Virtualmin, ces champs peuvent être utiles à d'autres fins.

Pour chaque champ, vous devez saisir les informations suivantes:

Nom de domaine
Un code court pour ce champ, qui est utilisé pour faire référence au champ dans les modèles et les scripts de post-création. Chaque champ doit avoir un nom de champ unique.

Étiquette
Le texte qui sera affiché à l'utilisateur lors de la saisie de la valeur de ce champ. Si l'étiquette contient un; , tout ce qui suit sera utilisé comme info-bulle qui apparaîtra lorsque vous passez la souris sur le champ de la page Modifier le serveur virtuel.

Type
Cela détermine quels types de valeurs peuvent être saisies pour ce champ. La plupart sont explicites, à l'exception de l'option Menu . Lorsque cette option est sélectionnée, vous devez entrer le chemin d'accès à un fichier sur le système Virtualmin dans la zone de texte adjacente. Chaque ligne du fichier doit contenir un code et du texte affiché pour une entrée dans le menu de ce champ personnalisé, séparés par des espaces.

Dans la liste des serveurs?
Si ce paramètre est défini sur Oui , une colonne pour ce champ personnalisé s'affiche sur la page Liste des serveurs virtuels .
