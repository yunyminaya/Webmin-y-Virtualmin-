Maximum Mongrel server processen voor Rails
Wanneer u Ruby on Rails scripts installeert kan de server administrator selecteren hoe de Mongrel server processen moeten starten om aanvragen af te handelen naar het pad van het script - als de Apache mod_proxy_balancer module is geintalleerd. Om te voorkomen dat er heel veel Mongrel server processen worden gestart die het systeem kunnen overbelasten, kan deze optie gebruikt worden om het aantal te limiteren die de domein administrators mogen configureren in Virtualmin.
