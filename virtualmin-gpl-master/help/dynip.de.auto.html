Dynamisches IP-Update
Auf dieser Seite können Sie Virtualmin einrichten, um Änderungen an der primären Netzwerkschnittstelle des Systems zu erkennen und einen dynamischen DNS-Eintrag zu aktualisieren, wenn sich dieser ändert. Dies ist nützlich, wenn Sie Virtualmin auf einem System mit einer Adresse ausführen, die sich gelegentlich ändert, z. B. in einer Umgebung mit virtuellen Maschinen wie Amazon EC2. Dies ist auch nützlich für Server, die über eine Verbindung ohne feste IP-Adresse mit dem Internet verbunden sind, z. B. ADSL oder Kabel.

Um diese Seite zu verwenden, müssen Sie einen dynamischen DNS-Dienst auswählen, den von Ihrem System für diesen Dienst verwendeten Hostnamen sowie die Anmeldedaten des Dienstes eingeben. Standardmäßig überprüft Virtualmin die der primären Netzwerkschnittstelle zugewiesene IP-Adresse (wie eth0 ), um nach Änderungen zu suchen. Wenn sich Ihr System jedoch hinter einem NAT-Router befindet, sollten Sie die im Internet sichtbare Adresse auswählen, damit die IP ermittelt wird dass der Rest des Internets wirklich sieht.

Wenn eine Änderung festgestellt wird, werden alle virtuellen Server mit gemeinsam genutzten IP-Adressen, die die alte Adresse verwenden, auf die neue aktualisiert. Diese Änderung wird sowohl in der Apache-Konfiguration als auch in den DNS-Einträgen vorgenommen.

Wenn der Dienst für externe Skripte ausgewählt ist, müssen Sie den vollständigen Pfad zu einem Skript in das nebenstehende Feld eingeben, das aufgerufen wird, wenn sich die IP-Adresse des Systems ändert. Die neue IP und der neue Hostname werden als Befehlszeilenparameter an das Skript übergeben.
