Пути прокси
Прокси позволяет сопоставить путь URL на веб-сайте виртуального сервера с одним или несколькими целевыми URL. Обычно это используется для того, чтобы приложения Ruby on Rails, обслуживаемые несколькими экземплярами Mongrel, были доступны по URL-пути на веб-сайте сервера. Его также можно использовать для предоставления доступа к другим веб-серверам, таким как Tomcat, по URL-адресу на вашем веб-сайте.

Например, вы можете сопоставить путь /radiant с URL-адресами http://localhost:3001 и http://localhost:3002 , которые являются портами для двух экземпляров Mongrel, обслуживающих одно и то же приложение, работающее в вашей системе Virtualmin.

Только системы, в mod_proxy_balancer установлен модуль Apache mod_proxy_balancer могут сопоставить путь с несколькими URL-адресами. Другие, которые поддерживают только обычный прокси-модуль Apache, могут перенаправлять путь только на один URL.
