Modo de ejecución de script PHP
Los modos de ejecución de scripts PHP definen cómo el servidor procesa el código PHP.

El modo deshabilitado significa que el servidor no ejecuta los scripts PHP, sino que los trata como archivos de texto sin formato. Este modo se puede utilizar cuando no se requiere procesamiento PHP o cuando se sirve código PHP como archivos descargables.

FPM es el modo preferido, ya que ofrece alta eficiencia y funciones avanzadas como la gestión adaptativa de procesos. Funciona como un servidor FastCGI independiente, ideal para sitios de alto tráfico debido a sus sólidas capacidades de rendimiento.

FCGI es útil para servidores donde FPM no está disponible. También admite configuraciones por host virtual, aunque generalmente no iguala a FPM en términos de escalabilidad y funciones de administración.

El modo CGI ejecuta cada script PHP como un proceso separado a través del protocolo CGI. Este modo es menos eficiente que FPM o FCGI debido a la sobrecarga de crear un nuevo proceso para cada solicitud. Rara vez se utiliza en configuraciones modernas debido a sus limitaciones de rendimiento.

mod_php integra PHP directamente en el servidor web Apache, permitiendo que los scripts PHP se ejecuten dentro del propio proceso del servidor. Sin embargo, debido a problemas de seguridad y aislamiento, además de la interrupción del soporte y las actualizaciones, ahora está obsoleto.
