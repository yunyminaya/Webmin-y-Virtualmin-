Añadir registro DNS SPF?
SPF, o Sender Policy Framework , es un medio para ayudar a reducir el correo no deseado mediante la aplicación de algunas restricciones sobre qué hosts pueden enviar correos que dicen ser de qué dominio. Debido a que la gran mayoría del spam se origina a partir de nombres de dominio falsificados, muchos creen que prevenir nombres de dominio falsificados en el spam reducirá el volumen total de spam. Si está habilitado, Virtualmin creará automáticamente un registro SPF en su servidor DNS. Este registro puede contener hosts específicos a los que se les permite enviar correo utilizando el nombre de dominio en cuestión (con el servidor en sí como el primer host, pero se puede enumerar cualquier número de hosts), o un genérico genérico, lo que significa El host puede enviar un correo que dice ser de este dominio. La segunda opción anula efectivamente el propósito de SPF, pero se proporciona para facilitar que los usuarios envíen correo "desde" su propio dominio a través de varios servidores de correo ISP.
