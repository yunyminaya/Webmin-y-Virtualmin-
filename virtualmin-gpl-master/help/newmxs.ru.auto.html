Вторичные почтовые серверы
Дополнительный почтовый сервер — это тот, который также может получать электронную почту для одного из виртуальных серверов, управляемых Virtualmin, но только как временный ретранслятор, а не конечный пункт назначения. Целесообразно настроить дополнительный сервер для интенсивно используемых почтовых доменов, чтобы он служил резервным или резервным местом на случай, если основной почтовый сервер не работает.

На этой странице вы можете выбрать другие серверы с установленным Virtualmin, которые будут использоваться в качестве дополнительных MX-серверов для новых почтовых доменов, созданных в этой системе. Прежде чем появятся какие-либо серверы, их необходимо сначала добавить в модуль Webmin Servers Index, в котором необходимо ввести root-логин и пароль для удаленного сервера.

Для каждого вторичного почтового сервера у вас есть возможность ввести другое имя хоста для использования в записи MX DNS вместо фактического имени хоста сервера, например, secmx.yourdomain.com . Однако убедитесь, что все, что вы вводите, является допустимым DNS-именем, которое разрешается на тот же сервер.

Если нажать кнопку Сохранить и применить , любые существующие виртуальные серверы, для которых включена электронная почта, будут добавлены (или удалены) из всех выбранных вторичных почтовых серверов. Кроме того, записи DNS MX будут добавлены (или удалены) из их доменов для ссылки на почтовые серверы.
