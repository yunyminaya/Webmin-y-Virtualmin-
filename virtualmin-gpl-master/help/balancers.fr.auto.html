Chemins d'accès proxy
Le proxy vous permet de mapper un chemin d'URL dans le site Web d'un serveur virtuel à une ou plusieurs URL de destination. Ceci est généralement utilisé pour rendre les applications Ruby on Rails servies par plusieurs instances Mongrel disponibles sous un chemin URL sur le site Web du serveur. Il peut également être utilisé pour rendre d'autres serveurs Web tels que Tomcat disponibles via un chemin URL sur votre site Web.

Par exemple, vous pouvez mapper le chemin /radiant aux URL http://localhost:3001 et http://localhost:3002 , qui sont les ports de deux instances Mongrel servant la même application exécutée sur votre système Virtualmin.

Seuls les systèmes sur lesquels le module Apache mod_proxy_balancer est installé peuvent mapper un chemin vers plusieurs URL. D'autres qui ne prennent en charge que le module proxy ordinaire d'Apache peuvent transférer un chemin vers une seule URL.
