Excluir backups antigos
Se você estiver fazendo backup em um diretório local ou bloco S3 cujo caminho é baseado no tempo do backup, essa opção poderá ser usada para excluir backups anteriores a alguns dias. Isso pode ser útil para reduzir a quantidade de espaço em disco necessária para armazenar dados antigos.

No entanto, isso só é utilizável quando o diretório de destino do backup é nomeado como /backup/%d-%m-%Y ou o nome do arquivo é nomeado como /backup/virtualmin-%d-%m-%Y.tar.gz . Isso é necessário para que o Virtualmin possa descobrir quais arquivos e diretórios foram criados pelo processo de backup.

Você também deve marcar a caixa Fazer substituições de horário no estilo do arquivo ou diretório , ou os códigos de data e hora no caminho do backup não serão convertidos quando o backup for executado.

A idade máxima do backup é inserida em dias, mas você pode usar frações como 0,25 por quatro horas.
