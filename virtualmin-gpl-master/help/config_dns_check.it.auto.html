Controlla resolv.conf per questo sistema?
Quando è impostato su Sì (per impostazione predefinita), Virtualmin controllerà il file di configurazione del client DNS /etc/resolv.conf per assicurarsi che sia configurato per utilizzare il server DNS locale per la risoluzione dei nomi. Ciò è necessario, poiché senza di esso e i domini DNS creati potrebbero non essere risolvibili dal sistema che esegue Virtualmin fino a quando non sono completamente registrati da un registrar DNS.

Se si è certi che ciò non sia necessario (forse perché si dispone di un'installazione automatica del server DNS secondario), questo può essere modificato in No per evitare errori di controllo della configurazione spuri.
