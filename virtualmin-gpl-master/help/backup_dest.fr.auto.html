Destination de sauvegarde
Ces champs contrôlent l'emplacement d'écriture de la sauvegarde Virtualmin. Vous pouvez choisir l'un des trois modes de destination :
Répertoire de fichiers locaux
Dans ce mode, la sauvegarde est simplement enregistrée dans un fichier sur le système exécutant Virtualmin. Si vous utilisez le format de sauvegarde à fichier d'archive unique , vous devez saisir un nom de fichier ici, comme /backup/domainname.tgz . Si vous utilisez l'un des formats à fichier unique par serveur , vous devez saisir un répertoire à la place, comme /backup .
Serveur FTP
Ce mode permet de télécharger le ou les fichiers de sauvegarde via le protocole FTP vers un autre serveur. Vous devez saisir le nom du serveur, le login, le mot de passe et le fichier ou le répertoire de destination.
Serveur SSH
Ceci est similaire au mode FTP, mais transfère la sauvegarde en utilisant le protocole SSH à la place.
Si le format de sauvegarde est un fichier unique, la destination que vous spécifiez ici doit être un fichier qui sera écrasé ou créé. Si vous créez un fichier par domaine, la destination doit être un répertoire qui existe déjà.

Si vous effectuez une sauvegarde sur un serveur SSH ou FTP sur un port non standard, vous pouvez le saisir dans le champ du serveur séparé du nom d'hôte par deux points, comme backupserver.yourdomain.com:2222 .

Si vous effectuez une sauvegarde sur un serveur SSH qui prend uniquement en charge l'authentification par certificat, le champ du mot de passe est facultatif. Cependant, l'utilisateur root du système doit avoir sa clé publique SSH dans le fichier authorized_keys de l'utilisateur de destination.
