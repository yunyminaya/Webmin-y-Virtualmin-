Cloudmin Services-Client
Auf dieser Seite können Sie Virtualmin so konfigurieren, dass eine Verbindung zu einem Cloudmin Services-Server hergestellt wird, sodass MySQL-Datenbanken, DNS-Zonen und andere Dienste auf Remote-Systeme ausgelagert werden können. Idealerweise sollte es verwendet werden, bevor virtuelle Server erstellt wurden, da das Mischen von lokal bereitgestellten und entfernten Datenbanken und Zonen ineffizient ist.

Die Felder, die ausgefüllt werden müssen, sind:

Cloudmin Services Master
Dieser Hostname eines Cloudmin-Computers mit installiertem Cloudmin Services-Plugin. Dies kann von Ihnen, Ihrem Hosting-Unternehmen oder VPS-Anbieter durchgeführt werden.

Port-Nummer
Dies sollte fast immer auf 10000 eingestellt sein, was der Standard-Webmin / Cloudmin-Port ist.

Benutzername für Server
Dieses Feld muss den Benutzernamen eines Systembesitzerkontos auf dem Cloudmin Services-Master enthalten, der zum Erstellen von Datenbanken, DNS-Zonen oder anderen Diensten berechtigt ist.
Funktionen zu hosten
In diesem Abschnitt können Sie auswählen, welche Virtualmin-Funktionen auf Remote-Systeme ausgelagert werden sollen. Es können nur diejenigen ausgewählt werden, die Ihr Konto auf dem Mastersystem bereitstellen darf.
