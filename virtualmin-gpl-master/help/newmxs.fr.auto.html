Serveurs de messagerie secondaires
Un serveur de messagerie secondaire est un serveur qui peut également recevoir des e-mails pour l'un des serveurs virtuels gérés par Virtualmin, mais uniquement comme relais temporaire plutôt que comme destination finale. Il est judicieux de configurer un serveur secondaire pour les domaines de messagerie très utilisés, pour servir de sauvegarde ou de stockage au cas où le serveur de messagerie principal serait en panne.

Sur cette page, vous pouvez sélectionner d'autres serveurs avec Virtualmin installé à utiliser comme serveurs MX secondaires pour les nouveaux domaines de messagerie créés sur ce système. Avant que les serveurs n'apparaissent, ils doivent d'abord être ajoutés au module Webmin Servers Index, dans lequel vous devez entrer le login et le mot de passe root pour le serveur distant.

Pour chaque serveur de messagerie secondaire, vous avez la possibilité d'entrer un nom d'hôte différent à utiliser dans l'enregistrement DNS MX, au lieu du nom d'hôte réel du serveur, tel que secmx.yourdomain.com . Cependant, assurez-vous que tout ce que vous entrez est un nom DNS valide qui se résout sur le même serveur.

Si vous cliquez sur le bouton Enregistrer et appliquer , tous les serveurs virtuels existants pour lesquels la messagerie est activée seront ajoutés (ou supprimés) à tous les serveurs de messagerie secondaires sélectionnés. De plus, les enregistrements DNS MX seront ajoutés (ou supprimés) à leurs domaines pour faire référence aux serveurs de messagerie.
