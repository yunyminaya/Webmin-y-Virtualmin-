Mover servidor virtual
Mover um servidor virtual altera a administração do domínio que tem controle sobre ele. As funções reais do servidor não são afetadas, mas agora podem ser gerenciadas por um administrador diferente e contam para quaisquer limites de domínios, aliases e bancos de dados para esse novo administrador.

Esta página pode ser usada para executar três tipos diferentes de operações, dependendo do tipo de servidor que você está editando e o que deseja fazer com ele. Eles são :

Mover um sub-servidor sob um pai diferente
Ao editar um sub-servidor, você pode selecionar um domínio pai diferente no menu para alterar sua propriedade. O site do servidor, o domínio DNS, as caixas de correio etc. não serão afetados, mas agora estará sob o controle do proprietário do novo pai selecionado. Seu diretório pessoal será movido para o novo pai, e todos os arquivos terão sua propriedade alterada para o usuário e grupo do novo pai.

Mover um servidor pai sob um pai diferente
Ao editar um servidor pai, você pode convertê-lo em um sub-servidor usando esta página. Isso deve ser feito com cuidado, pois algumas configurações existentes apenas para servidores-pai (como senha, cotas e limites) serão perdidas, pois agora são herdadas do novo pai. Além disso, qualquer usuário Unix, grupo Unix ou usuário Webmin associado ao domínio será excluído.

Converter um sub-servidor em um servidor pai
Ao editar um sub-servidor, você tem a opção de convertê-lo em um servidor pai pertencente a um novo administrador de servidor. Um nome de usuário e senha para o novo administrador devem ser inseridos. Após a conclusão da movimentação, todos os arquivos no novo diretório inicial do servidor pertencerão ao novo usuário. As cotas de disco, os limites de largura de banda e os limites de proprietário do servidor para o servidor movido serão copiados do pai antigo, mas poderão ser editados posteriormente separadamente.

Alterando o destino de um alias
Se o servidor selecionado for um alias de outro servidor, este formulário poderá ser usado para apontá-lo para um destino diferente. Após a conclusão da movimentação, o acesso da Web ao domínio retornará o conteúdo do novo destino e o email para o domínio será encaminhado para as caixas de correio do novo destino.
