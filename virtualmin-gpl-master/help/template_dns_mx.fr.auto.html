Nom d'hôte pour l'enregistrement MX
Par défaut, l'enregistrement DNS MX (qui contrôle l'endroit où les e-mails sont livrés) pour les nouveaux domaines sera mail.${DOM}, qui se verra attribuer la même adresse IP que le site Web du serveur virtuel. Cependant, vous pouvez utiliser ce champ pour spécifier un nom d'hôte de serveur de messagerie différent - il est généralement utilisé pour avoir un enregistrement MX cohérent sur tous les domaines. Assurez-vous d'entrer un nom d'hôte qui correspond à votre système Virtualmin, sinon le courrier électronique ne sera pas livré.

Vous pouvez également utiliser les variables standard de Virtualmin comme ${DOM} pour spécifier un enregistrement MX avec un nom différent dans le domaine en cours de création.
