Réallouer les UID et GID?
Lorsque cette option est définie sur Oui et que vous restaurez un serveur qui n'existe pas actuellement sur le système, les ID utilisateur et groupe Unix pour l'administrateur du serveur et les utilisateurs de messagerie / FTP seront réaffectés pour utiliser des ID gratuits sur votre système. C'est généralement ce que vous voulez car cela évite les conflits avec les ID existants sur votre système.

La sélection de Non indique à Virtualmin d'utiliser les ID utilisés lors de la création de la sauvegarde. Cela doit être utilisé si d'autres fichiers existent toujours sur le système avec les anciens ID et si ces ID n'ont pas été alloués à d'autres utilisateurs ou groupes depuis la sauvegarde.
