Configurare il processo Cron Webalizer per ciascun server virtuale?
Quando questa opzione è impostata su Sì (come per impostazione predefinita), Virtualmin creerà un processo Cron separato per ogni server virtuale in cui Webalizer è abilitato, al fine di generare un report dai suoi registri su base giornaliera.

Tuttavia, su un server con un gran numero di domini potresti invece voler cambiare questa opzione in No e invece programmare manualmente il comando /usr/libexec/webmin/virtual-server/run-all-webalizer.pl usando Cron. Ciò assicurerà che Webalizer venga eseguito per ogni dominio a turno anziché potenzialmente avere più domini elaborati contemporaneamente e ti darà un maggiore controllo esattamente al termine dell'elaborazione.
