Crear subdominio virtual
Este formulario le permite crear un nuevo subdominio virtual, que es propiedad de un inicio de sesión de administración existente. Un subdominio es similar a un sub-servidor, ya que comparte la cuota de disco, los límites de ancho de banda y los límites de usuarios, alias y otros objetos con su padre. Sin embargo, el nombre del subdominio debe estar debajo del nombre del dominio principal, utiliza un subdirectorio del directorio public_html del padre para su sitio web y se comparten los archivos de registro.

Al crear el servidor, debe ingresar al menos el nombre de dominio y una descripción. Si se definen varias plantillas de servidor en el sistema, puede seleccionar una del menú: las plantillas determinan varios valores predeterminados para el nuevo servidor, como la configuración del sitio web, los registros DNS, etc.

Para seleccionar qué funciones están habilitadas para el servidor, use las casillas de verificación en la segunda parte de la sección Opciones avanzadas . Los subdominios tienen un conjunto mucho más pequeño de características disponibles en comparación con los servidores secundarios o los servidores de nivel superior.

En la parte inferior de la sección Opciones avanzadas , puede seleccionar que todo el tráfico web a este nuevo sitio se reenvíe a otra URL. También puede seleccionar que el nuevo servidor virtual use sus propias direcciones IP privadas, que se requieren para el alojamiento SSL y FTP anónimo.
