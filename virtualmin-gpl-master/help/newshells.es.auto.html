Conchas personalizadas
Esta página le permite personalizar los shells de Unix disponibles para usuarios de buzones y administradores de servidores virtuales. El shell es utilizado por servidores como ProFTPd y SSHd para controlar los tipos de inicio de sesión permitidos para cada usuario, como el correo electrónico, FTP y acceso remoto al shell.

Cuando se selecciona los valores predeterminados integrados de Virtualmin, Virtualmin determina automáticamente los shells disponibles, lo que generalmente es satisfactorio para la mayoría de los sistemas. Si desea modificarlos, seleccione Conchas personalizadas a continuación y realice cambios en la tabla.

Para cada shell, se pueden editar los siguientes campos:

Habilitado?
Esta casilla de verificación determina si el shell es visible al crear o editar usuarios.
Camino a la cáscara
Debe contener la ruta completa al comando de shell, como /bin/sh o /bin/false .
Descripción
Este es el título que aparecerá en el menú cuando se seleccione el shell.
¿Administración?
Marque esta casilla si desea que el shell esté disponible para los usuarios de Unix del administrador del servidor virtual.
¿Buzón?
Marque esta casilla si desea que el shell esté disponible para usuarios de correo / FTP.
¿Defecto?
Exactamente un shell para administradores y uno para buzones debe tener esta entrada marcada, lo que indica que es el valor predeterminado para los usuarios de Unix de ese tipo.
Acceso
Este menú indica a Virtualmin qué tipo de acceso otorga este shell. Al agregar el suyo, debe seleccionar si permite FTP y / o SSH o no.
