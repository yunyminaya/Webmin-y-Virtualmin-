Crea un sub-server virtuale
Questo modulo consente di creare un nuovo sub-server virtuale, di proprietà di un accesso amministrativo esistente. Un sotto-server condivide anche la quota del disco, i limiti di larghezza di banda e i limiti di utenti, alias e altri oggetti con il suo genitore.

Quando si crea il server, è necessario inserire almeno il nome di dominio e una descrizione. Se sul sistema sono definiti più modelli di server, è possibile selezionarne uno dal menu: i modelli determinano varie impostazioni predefinite per il nuovo server, come la configurazione del sito Web, i record DNS e così via.

Nella sezione Opzioni avanzate è possibile selezionare un nome di database iniziale diverso, anziché utilizzare il valore predefinito derivato dal nome di dominio. I nomi utente e di gruppo non possono essere modificati, poiché sono sempre ereditati dal server di appartenenza.

Per selezionare quali funzioni sono abilitate per il server, utilizzare le caselle di controllo nella seconda parte della sezione Opzioni avanzate . Alcune funzionalità dipendono da altre e non possono essere selezionate da sole - ad esempio, i report di Webalizer sulle statistiche dei siti Web dipendono dall'avere effettivamente un sito Web.

Nella parte inferiore della sezione Opzioni avanzate è possibile selezionare per inoltrare tutto il traffico Web a questo nuovo sito a un altro URL e inoltrare tutta la posta elettronica. Puoi anche scegliere di fare in modo che il nuovo server virtuale utilizzi i propri indirizzi IP privati, richiesti per SSL e hosting FTP anonimo.
