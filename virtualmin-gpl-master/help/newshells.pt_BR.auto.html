Shell personalizados
Esta página permite que você personalize os shells do Unix disponíveis para usuários de caixas de correio e administradores de servidores virtuais. O shell é usado por servidores como ProFTPd e SSHd para controlar os tipos de logins permitidos para cada usuário, como email, FTP e acesso remoto ao shell.

Quando os padrões internos do Virtualmin são selecionados, os shells disponíveis são determinados automaticamente pelo Virtualmin, que normalmente é satisfatório para a maioria dos sistemas. Se você deseja modificá-los, selecione Shell personalizados abaixo e faça alterações na tabela.

Para cada shell, os seguintes campos podem ser editados:

Ativado?
Essa caixa de seleção determina se o shell está visível ao criar ou editar usuários.
Caminho para o shell
Isso deve conter o caminho completo para o comando do shell, como /bin/sh ou /bin/false .
Descrição
Este é o título que aparecerá no menu quando o shell for selecionado.
Admin?
Marque esta caixa se desejar que o shell seja disponibilizado para usuários Unix do administrador do servidor virtual.
Caixa de correio?
Marque esta caixa se desejar que o shell seja disponibilizado para usuários de email / FTP.
Padrão?
Exatamente um shell para administradores e um para caixas de correio devem ter essa entrada marcada, indicando que é o padrão para usuários Unix desse tipo.
Acesso
Este menu indica ao Virtualmin que tipo de acesso é concedido por esse shell. Ao adicionar o seu próprio, você deve selecionar se ele permite FTP e / ou SSH ou não.
