Subprocesos máximos del servicio PHP
Esta opción establece el límite en la cantidad de solicitudes simultáneas que se atenderán; cuando los scripts PHP para este dominio se configuren a través de FPM, la cantidad de procesos secundarios se creará cuando la opción pm esté configurada en static y la cantidad máxima de procesos secundarios. potencialmente podría crearse cuando la opción pm está configurada como dynamic.

Cuando los scripts PHP para la configuración de este dominio se ejecutan a través de FCGI, la administración de procesos secundarios de PHP siempre debe estar deshabilitada, lo que solo enrutará una solicitud a la vez a los procesos de aplicación que ha generado. De forma predeterminada, y con la configuración de la variable de entorno PHP_FCGI_CHILDREN=0 , la gestión de procesos secundarios de PHP está deshabilitada.
